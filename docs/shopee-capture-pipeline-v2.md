# Shopee Capture Pipeline v2 — ระบบเก็บข้อมูล/รูปที่เสถียร

จัดทำ: 2 กรกฎาคม 2026
แทนที่: วิธี bookmarklet ใน `marketplace-user-assisted-capture.md` (ยังใช้เป็น fallback ได้ แต่ไม่ใช่วิธีหลัก)

## ทำไมวิธี bookmarklet ถึงไม่เสถียร (บทเรียน)

1. Shopee เป็น React SPA — ข้อมูลจริงมาจาก API JSON ไม่ได้อยู่ใน HTML; bookmarklet อ่าน DOM ณ วินาทีที่กด ส่วนที่ lazy-load ยังไม่มาก็หายไป
2. รูป gallery โหลดแบบ lazy เห็นแค่รูปแรก ๆ และการดึง binary ข้าม origin โดน CORS block — ได้แต่ URL thumbnail ความละเอียดต่ำ
3. Shopee ใช้ class name แบบสุ่มและเปลี่ยนบ่อย — script ที่อิง selector พังทุกครั้งที่ UI อัปเดต

## เกณฑ์ตัดสินวิธีที่ดี

เสถียร (ไม่พังเมื่อ Shopee เปลี่ยน UI) · ทำได้ตลอด (ไม่โดนบล็อก ไม่เสี่ยงบัญชี) · ข้อมูลถูก (ตรงกับที่หน้าแสดงจริง + ตรวจย้อนได้) · แรงน้อย (คนเดียวทำไหว)

## สถาปัตยกรรม 3 ชั้น

### ชั้น 1 (วิธีหลัก): Claude in Chrome

ผู้ใช้เปิด Chrome ที่ล็อกอิน Shopee ตามปกติ → Claude อ่านหน้าผ่าน extension:

- อ่านข้อมูลจาก **accessibility tree** ไม่ใช่ class name → Shopee เปลี่ยน UI ก็ยังอ่านได้ (แก้ปัญหาความไม่เสถียรที่ราก)
- เก็บ URL รูปทั้งหมดจาก CDN (`down-th.img.susercontent.com/file/...`) ระหว่างเปิด gallery → **ดาวน์โหลดรูปทีหลังด้วย PowerShell นอก browser** (CDN สาธารณะ ไม่ติด CORS ไม่ติด login) — แก้ปัญหาดึงรูปที่ราก
- แคปหน้าจอเป็น screenshot evidence เข้าช่อง `screenshot_file` ของ ledger
- Claude กรอก ledger + สรุปข้อมูลเองทันที ไม่ต้องมีไฟล์ JSON คั่นกลาง

จังหวะทำงาน (คงกติกา B+D เดิม): 1 คำค้น/รอบ → เปิดหน้าสินค้าทีละหน้า ความเร็วมนุษย์ → รอบละไม่เกิน 10–15 หน้า → เจอ verification เมื่อไหร่หยุดทันที พักแล้วค่อยรอบใหม่

ติดตั้งครั้งเดียว: Chrome Web Store → "Claude in Chrome" → sign in บัญชี Claude เดียวกับที่ใช้ Claude Code

### ชั้น 2 (fallback เสถียรสุด): Screenshot drop

ใช้เมื่อ Shopee เด้ง captcha, ใช้มือถือ, หรือ extension มีปัญหา:

1. ผู้ใช้แคปหน้าสินค้า (Win+Shift+S เลื่อนแคปให้ครบ: ราคา/rating/จำนวนรีวิว/ตัวเลือก BTU/ร้าน/ประกัน) หรือแคปจากแอปมือถือ
2. วางรูปในแชท Claude หรือใส่โฟลเดอร์ `research/inbox/screenshots/`
3. Claude อ่านภาพ → กรอกข้อมูลเข้า ledger; ไฟล์แคปคือ evidence โดยตัวมันเอง

ข้อดีเด็ดขาด: ไม่มีทางถูกบล็อก เพราะเป็นการใช้งานปกติ 100% / ข้อเสีย: ช้ากว่า (~2 นาที/รุ่น) เหมาะกับ 5–15 รุ่น/รอบ ซึ่งตรงกับจังหวะ B+D อยู่แล้ว

### ชั้น 3 (ระยะยาว): Affiliate API / datafeed

เมื่อบัญชี Shopee Affiliate อนุมัติ (หรือผ่าน Involve Asia / Accesstrade):

- ดึงราคา/สถานะสินค้า/ลิงก์ affiliate เชิงโปรแกรม → งานอัปเดตราคารายเดือนกลายเป็นกึ่งอัตโนมัติ
- ข้อมูล conversion จริงจาก dashboard = สัญญาณความนิยมที่แม่นกว่ายอดขายหน้าเว็บ
- ชั้น 1/2 ยังจำเป็นสำหรับ: ธีมรีวิวผู้ซื้อ, ภาพ, ข้อมูลหน้าร้าน (ประกัน/ติดตั้ง)

## กติกาความถูกต้องของข้อมูล (ทุกชั้น)

- บันทึกเฉพาะสิ่งที่มองเห็นจริง + วันที่เก็บ เสมอ (กติกา B+D เดิม)
- ยอดขายไม่แสดง → `sold_count_visible=false` ใช้ ratings/comments แทน ห้ามเดา
- รูปทุกไฟล์บันทึก source URL + วันเก็บ และเช็ก hash กันไฟล์ซ้ำ (กติกาเดิม)
- ข้อมูลเกิน 14–30 วัน ต้องเก็บซ้ำก่อนใช้เขียน
- ห้ามใช้ตัวเลขจาก AI/ChatGPT/ภาพ infographic ภายนอกเป็นหลักฐาน — ใช้ได้แค่เป็น candidate list (บทเรียน 2 ก.ค. 2026: ตัวเลขที่ AI สร้างขัดแย้งกันเองและปนสกุลเงินต่างประเทศ)

## Image pipeline ใหม่ (แก้ปัญหารูปโดยเฉพาะ)

1. ระหว่าง capture: เก็บ URL รูปเต็มทุกรูปใน gallery (คลิกไล่ดูให้ Shopee โหลดครบ)
2. Claude ดาวน์โหลดนอก browser ด้วย PowerShell (ใส่ browser User-Agent) ลงโฟลเดอร์ชั่วคราว
3. ตรวจ: ความละเอียดขั้นต่ำ, ไม่ใช่ thumbnail, hash ไม่ซ้ำของเดิม
4. แปลง/บีบอัด → ตั้งชื่อตามแพทเทิร์นเดิม (`{productKey}-hero.jpg`, `-product-clean.jpg`, ...) → เข้า `public/images/...`
5. จด source URL + capture date ใน ledger/notes
