## บทสรุปผู้บริหาร

Friendsay ยืนอยู่ระหว่างสองค่ายที่ไม่มีใครในไทยเชื่อมได้: ค่าย "ราคา" (Priceza/BigGo/iPrice) มีข้อมูลแต่ไม่มีคำฟันธง, ค่าย "รีวิว" (Beartai/Droidsans/Mercular) มีคำฟันธงแต่มีอคติสปอนเซอร์หรือมีของขายเอง. Friendsay มีของหลักครบแล้ว (roundup ฟันธง, /th/vs/ programmatic, catalog 6 หมวด, trust strip, B+D anti-fabrication, ledger เริ่มต้น) จึงไม่ต้องสร้างฐานใหม่ แต่ต้อง "เก็บเกี่ยว" 3 อย่างที่ยังรั่ว: (1) traffic ฟรี — ยังไม่ submit Search Console และยังไม่มีหน้าตัวตน/ผู้เขียน ทำให้ Google ไม่เชื่อ E-E-A-T; (2) trust ที่หาเงินไม่ได้ในไทย — ยังไม่กล้าออก "รุ่นที่ควรเลี่ยง" และยังไม่ประกาศจุดยืนความเป็นกลางแบบชูธง; (3) retention — ยังไม่มี loop ดึงคนกลับ (trending/alert). แผนนี้จัดลำดับให้ทำของฟรี-ผลไว-เจ้าของคนเดียวไหวก่อน (P0), ต่อยอด SEO/engagement จากของที่มี (P1), แล้วค่อยลงทุน pipeline ราคา/retention ที่ติดข้อมูล (P2-P3). ทุกข้อเคารพข้อจำกัด "ไม่มีสินค้าจริง" — ไม่มีข้อไหนต้องแกล้งจับของทดสอบ.


## จุดยืนที่ไม่มีเว็บไทยเจ้าไหนยึด (positioning)

ช่องว่างที่ไม่มีเว็บไทยเจ้าไหนยึด = "เว็บฟันธงที่พิสูจน์ย้อนได้และเป็นกลางเพราะไม่มีของขาย". สามขาพร้อมกัน: (ก) VERDICT — บอกว่ารุ่นไหนควรซื้อ/ควรเลี่ยง (Priceza/BigGo บอกแค่ 'ที่ไหนถูก' ไม่บอก 'อันไหนดีกว่า'); (ข) EVIDENCE — ทุกคำฟันธงมีวันเก็บข้อมูล ตารางเทียบ ledger ราคา และ changelog ให้ตรวจย้อน (สื่อรีวิวไทยไม่มีชั้นข้อมูลนี้และมีสปอนเซอร์แฝง); (ค) NEUTRALITY — Mercular ฟันธงได้แต่จัดอันดับได้เฉพาะของที่ตัวเองสต็อก ส่วน Friendsay ไม่มีสต็อกจึงแนะนำได้ทุกยี่ห้ออย่างที่ร้านทำไม่ได้. จุดยืนหนึ่งประโยคที่ควรเป็นสโลแกนถาวร: "เราไม่มีของขาย เราถึงเชียร์ได้ทุกยี่ห้อ และกล้าบอกตัวที่ไม่ควรซื้อ." คู่แข่งไทยแต่ละเจ้าลอกได้ทีละขา แต่ลอกครบสามขาพร้อมกันไม่ได้เพราะโครงสร้างรายได้ขัดกันเอง.

### รายละเอียดรายข้อ


#### [P0] Submit Search Console + sitemap + internal-link mesh ข้าม vs/catalog/roundup
- **คืออะไร:** ส่ง sitemap เข้า Google Search Console (ยังไม่ทำ) แล้ววางตาข่ายลิงก์ภายในแบบ programmatic: ทุกหน้า /th/vs/ ลิงก์ไป roundup + catalog ของหมวดนั้น, ทุก catalog item ลิงก์เข้าคู่เทียบยอดนิยม, roundup ลิงก์ลง vs. ทำให้ 141 หน้าเสริมกันเป็น long-tail SEO engine
- **ต้นแบบ:** versus.com, GSMArena, iPrice Thailand
- **Friendsay ขาดอะไร:** ยังไม่ submit Search Console เลย = Google ยังไม่รู้จักโครงสร้างเว็บเต็ม และหน้า 141 หน้ายังลอยแยกกัน ไม่มี mesh ดึงกันเอง
- **effort:** low · **ติดที่:** ไม่มี — ทำได้ทันที (owner login + verify domain)
- **KPI:** จำนวนหน้า indexed ใน GSC, organic impressions/clicks รายสัปดาห์, avg internal links/page

#### [P0] หน้าตัวตน/ผู้เขียน + วิธีการทำงาน (E-E-A-T) พร้อม author schema
- **คืออะไร:** ต่อยอดจาก about.astro/editorial-policy ที่มี ให้เป็นหน้า 'ใครอยู่เบื้องหลัง Friendsay + เราคัด/ตรวจยังไง' มี byline ประจำทุก roundup/รีวิว + JSON-LD author/Organization. ยึดความจริง: ระบุตรงว่าใช้ข้อมูลจริง+AI ช่วยวิเคราะห์ ไม่แกล้งเป็นห้องแลบจับของ
- **ต้นแบบ:** Wirecutter (NYT), RTINGS.com, Wongnai
- **Friendsay ขาดอะไร:** ระบุใน current-state ว่า 'ยังไม่มีหน้าตัวตนทีม/ผู้เขียน' — ขาด E-E-A-T ที่ Google ใช้ตัดสิน review content ตั้งแต่ Helpful Content update
- **effort:** med · **ติดที่:** เจ้าของ — ต้องตัดสินใจว่าจะเปิดตัวตน/persona แบรนด์แบบไหน (ต้องจริง ห้ามปลอมชื่อผู้ทดสอบ)
- **KPI:** มี author schema ผ่าน Rich Results Test, organic CTR ของหน้ารีวิว, ranking หน้า 'ยี่ห้อไหนดี'

#### [P0] บล็อก 'รุ่นที่ควรเลี่ยง / ยังไม่ควรซื้อ' ในทุก roundup
- **คืออะไร:** เพิ่ม section บังคับทุก roundup: 1-2 รุ่นที่ไม่แนะนำ + เหตุผลอ้างหลักฐาน (สเปกหลอก/ราคาแพงเกินคุณภาพ/เคลมเกินจริงที่ debug แล้ว) โดยไม่ใส่ลิงก์ affiliate ให้ตัวที่เตือน. ต่อยอดจาก 'จุดระวัง' รายตัวที่มี ยกระดับเป็นคำตัดสิน binary ระดับ roundup
- **ต้นแบบ:** Which? (UK), Wirecutter (NYT)
- **Friendsay ขาดอะไร:** ปัจจุบันมีแค่ 'จุดระวัง' ต่อ pick แต่ไม่กล้าออกลิสต์ 'อย่าซื้อ' ระดับหมวด — เสีย costly-signal trust ที่เว็บ affiliate ไทยแทบไม่มีใครทำ
- **effort:** low · **ติดที่:** ไม่มีด้านเทคนิค — เป็น editorial guideline (ต้องมีหลักฐานหนุนกันความเสี่ยงถูกแบรนด์ทัก)
- **KPI:** % roundup ที่มี ≥1 รุ่นควรเลี่ยง, time-on-page ของ section, scroll-depth ถึงบล็อกนี้

#### [P0] หน้าจุดยืน 'เราไม่มีของขาย' + badge ความเป็นกลางบนทุก roundup
- **คืออะไร:** หน้าถาวร + แบดจ์เล็กติดหัว roundup/vs: 'Friendsay ไม่ได้สต็อกสินค้าเอง จึงจัดอันดับได้ทุกยี่ห้อ และเปิดเผยลิงก์ affiliate ทุกจุด'. เป็นการโจมตีจุดที่ Mercular (ร้าน) และสื่อรับสปอนเซอร์ทำไม่ได้เชิงโครงสร้าง
- **ต้นแบบ:** Mercular.com, Honey / PayPal (anti-pattern), Pantip (CR/SR)
- **Friendsay ขาดอะไร:** มี affiliate-disclosure แล้วแต่ยังไม่ได้แปลง 'ความเป็นกลางเพราะไม่มีของขาย' เป็นจุดขาย/แบดจ์ชูธง — เป็น differentiator ที่ยังไม่ถูกใช้
- **effort:** low · **ติดที่:** ไม่มี — copywriting + policy consistency
- **KPI:** CTR badge->หน้าจุดยืน, brand-search 'friendsay' รายเดือน, bounce rate หน้า roundup

#### [P1] บล็อก 'ทำไม A ดีกว่า B' เป็น bullet ส่วนต่าง ดัก featured snippet
- **คืออะไร:** ต่อยอด /th/vs/ ที่มี radar+ป้าย N จุด ให้เจนบล็อก bullet อัตโนมัติจากตารางสเปก: ข้อที่ A ชนะขึ้นพร้อมค่าต่าง ('BTU สูงกว่า 12000 vs 9000', 'เสียงเบากว่า 3 dB') แยกสองฝั่ง เขียนเป็นถ้อยคำที่ Google หยิบทำ snippet คีย์ 'A vs B'
- **ต้นแบบ:** versus.com, GSMArena
- **Friendsay ขาดอะไร:** มีข้อมูลเทียบ+radar แล้วแต่ยังไม่มีบล็อกข้อความส่วนต่างที่ optimize สำหรับ featured snippet — พลาด zero-click ranking
- **effort:** med · **ติดที่:** ข้อมูล — ต้องกำหนดกฎ 'มาก/น้อย=ดีกว่า' ต่อสเปก (บางค่าน้อยดีกว่า) และสเปกต้องเป็นตัวเลขสะอาด
- **KPI:** จำนวน featured snippet ที่ครอง, organic traffic หน้า /th/vs/, ranking คีย์ 'X vs Y'

#### [P1] Chip 'ราคานี้ ถูก/ปกติ/แพง' + sparkline จาก ledger
- **คืออะไร:** เมื่อ ledger สะสมถึงเกณฑ์ (เช่น ≥8 จุด/รุ่น) แสดงแบดจ์ตัดสินราคาปัจจุบันเทียบต่ำสุด/เฉลี่ย 90 วัน + กราฟจิ๋วใต้ปุ่มซื้อ. ก่อนถึงเกณฑ์ ให้ formalize cadence เก็บ snapshot ราคารายวันจาก Chrome เจ้าของเป็น routine ตายตัว
- **ต้นแบบ:** CamelCamelCamel, Keepa, BigGo Thailand, Priceza.com
- **Friendsay ขาดอะไร:** current-state ระบุ ledger ยังบาง (<3 จุด/รุ่น) และ 'chip ราคาดี/แพง รอ ledger สะสม' — กลไกที่ transferable ที่สุดยังใช้ไม่ได้เพราะข้อมูลไม่พอ
- **effort:** high · **ติดที่:** ข้อมูล+เจ้าของ — ต้องมี cadence เก็บราคาต่อเนื่อง (แมนนวล Shopee ผ่าน Chrome) จนหนาพอ chip ถึงจะซื่อสัตย์
- **KPI:** จำนวนจุดราคาเฉลี่ย/รุ่น, % รุ่นที่แสดง chip ได้, CTR ปุ่มซื้อเมื่อมี chip 'ถูก' vs ไม่มี

#### [P1] สไลเดอร์ปรับความสำคัญบนเครื่องมือเทียบ (เน้นราคา/เย็นไว/เงียบ/ประหยัดไฟ)
- **คืออะไร:** ต่อยอดเครื่องมือเทียบ (เลือก 2-4 รุ่น radar+verdict) ที่มี ให้ผู้ใช้เลื่อนน้ำหนัก use-case แล้วผู้ชนะ/verdict สลับเรียลไทม์ฝั่ง client. ยอมรับว่า 'ดีที่สุด' ขึ้นกับความต้องการแต่ละคน
- **ต้นแบบ:** versus.com, RTINGS.com
- **Friendsay ขาดอะไร:** เครื่องมือเทียบปัจจุบันให้ verdict คงที่ ยังไม่ให้ผู้ใช้ถ่วงน้ำหนักตามที่ตัวเองแคร์ — engagement ต่ำกว่าที่ควร
- **effort:** med · **ติดที่:** ข้อมูล+เทคนิค — ต้องมี scoring model ต่อหมวด และ JS ต้องรอด View Transitions client-nav (re-init หลัง navigate)
- **KPI:** เวลาบนหน้าเทียบ, สัดส่วนคนที่ขยับสไลเดอร์, คลิกซื้อจากหน้าเทียบ

#### [P1] วิดเจ็ต 'สินค้ามาแรงสัปดาห์นี้' จาก pageview จริง (GA4)
- **คืออะไร:** นับ pageview ต่อ slug จาก GA4 ที่มีอยู่ ออกวิดเจ็ต/บล็อก 'มาแรงรายสัปดาห์' ต่อหมวดแบบอัปเดตอัตโนมัติ วัดดีมานด์จริงจากพฤติกรรม (ปั่นยาก) เป็นคอนเทนต์ที่คนกลับมาดู
- **ต้นแบบ:** GSMArena, Kakaku.com
- **Friendsay ขาดอะไร:** มี GA4 แล้วแต่ยังไม่แปลงเป็น trending signal บนหน้า — ขาด loop 'ตลาดกำลังมองอะไร' ที่ดึง return visit
- **effort:** med · **ติดที่:** ข้อมูล+เทคนิค — ต้องดึง GA4 Data API เข้า build/edge และ widget ต้องรอด client-nav
- **KPI:** return-visitor rate, CTR วิดเจ็ต trending, pageview ของรุ่นที่ติด trending

#### [P1] Chip เรตติ้ง 'ระบุแหล่ง + จำนวนรีวิว' ต่อสินค้า
- **คืออะไร:** ต่อยอด trust strip (นับรีวิวรวม) ลงมาระดับการ์ดสินค้า: '⭐4.3 · จาก 2,021 รีวิว Shopee' ระบุแหล่งชัด เพื่อ out-transparent Priceza ที่โชว์เรตติ้งรวมโดยไม่บอกที่มา
- **ต้นแบบ:** Priceza.com, GSMArena, Kakaku.com
- **Friendsay ขาดอะไร:** trust strip มีเลขรวม แต่การ์ดสินค้ายังไม่โชว์เรตติ้ง+sample size ที่ระบุแหล่งต่อ SKU — เป็น social proof ราคาถูกที่ยังไม่ได้ใช้
- **effort:** low · **ติดที่:** ข้อมูล — ต้องเก็บ rating+review count ต่อ SKU (แมนนวลตอนเก็บราคาอยู่แล้ว) และ normalize
- **KPI:** CTR การ์ดที่มี chip เรตติ้ง vs ไม่มี, conversion ปุ่มซื้อ

#### [P2] หน้า hub ดีลตามปฏิทินไทย + จับโปรลวง 'ขึ้นก่อนลด'
- **คืออะไร:** landing รวมดีลเด่นช่วง 11.11/12.12/Songkran อัปเดตช่วงแคมเปญ + ใช้ ledger เปิดโปงรุ่นที่ดันราคาขึ้นก่อนติดป้ายลด. ยึดจุดยืน 'อยู่ข้างผู้ซื้อ' ที่ Priceza/iPrice ให้ไม่ได้
- **ต้นแบบ:** Slickdeals, BigGo Thailand
- **Friendsay ขาดอะไร:** ยังไม่มี seasonal hub และยังไม่ใช้ ledger จับ fake-discount — พลาด traffic peak เทศกาลที่คาดเดาได้
- **effort:** med · **ติดที่:** ข้อมูล — ต้องมี snapshot ก่อน-หลังแคมเปญ (ได้บางส่วนจาก ledger ถ้าเก็บทัน) + เจ้าของเตรียมล่วงหน้า
- **KPI:** traffic ช่วงแคมเปญ vs ปกติ, จำนวนรุ่นที่ตรวจพบโปรลวง, คลิก affiliate ช่วงเทศกาล

#### [P2] บล็อก 'โค้ดส่วนลดวันนี้' ท้าย funnel
- **คืออะไร:** ใต้ปุ่มซื้อแต่ละรุ่น เพิ่มกล่องโค้ด Shopee/Lazada/TikTok ล่าสุด (copy ได้) ดัก intent จังหวะจ่ายเงินที่ conversion สูง โดยไม่กั๊กโค้ดที่ดีกว่า (ตรงข้าม Honey)
- **ต้นแบบ:** iPrice Thailand, ShopBack Thailand
- **Friendsay ขาดอะไร:** ยังไม่มีชั้นโค้ดส่วนลด — พลาด conversion ปลายทางที่ค่าคอมสูง
- **effort:** low · **ติดที่:** ข้อมูล — ต้องเข้าถึงฟีดคูปองจาก affiliate network (Involve Asia/Accesstrade) หรืออัปเดตมือรายสัปดาห์
- **KPI:** conversion rate หน้าที่มีกล่องโค้ด, จำนวน copy โค้ด, ยอด affiliate ผ่านโค้ด

#### [P2] แจ้งเตือนราคาลดผ่าน LINE OA/อีเมล แบบไม่ต้องล็อกอิน
- **คืออะไร:** ปุ่ม 'เตือนเมื่อราคาถึง ___ บาท' หรือ 'ลด ≥15% จากเฉลี่ย' เหนือ/ใต้ปุ่มซื้อ เก็บ LINE/email แล้วยิงกลับพร้อมลิงก์ affiliate เมื่อถึงเป้า — retention loop ที่เว็บรีวิวไทยไม่มี
- **ต้นแบบ:** CamelCamelCamel, Keepa, Slickdeals, idealo
- **Friendsay ขาดอะไร:** current-state ระบุ 'ยังไม่มี LINE alert' — ไม่มีกลไกดึงคนกลับเอง พึ่ง organic ล้วน
- **effort:** high · **ติดที่:** ข้อมูล+เจ้าของ — ต้องมี price feed สด (ผูกกับ ledger ข้อ P1) + LINE Messaging API/อีเมล + backend เก็บ watch; เจ้าของต้องอนุมัติ infra
- **KPI:** จำนวนคนตั้ง alert, click-back rate จากข้อความเตือน, conversion จาก alert

#### [P3] ตัวช่วยเลือกด้วยสเปก (Finder) ต่อหมวด
- **คืออะไร:** หน้า 'เลือก[สินค้า]ตามงบ/คุณสมบัติ' กรองด้วยช่วงราคา + 2-3 สเปกหลัก (BTU, ประหยัดไฟ, เสียง) แล้วลิงก์เข้ารีวิว/เทียบ — เปลี่ยน catalog ให้เป็นเครื่องมือเลือกซื้อ ดักคนที่รู้ความต้องการแต่ยังไม่รู้รุ่น
- **ต้นแบบ:** GSMArena, versus.com, RTINGS.com
- **Friendsay ขาดอะไร:** catalog 6 หมวด live แล้วแต่ยังไม่มี spec-filter finder — คนกรองเองไม่ได้
- **effort:** med · **ติดที่:** ข้อมูล+เทคนิค — ต้องมีฐานสเปก structured ต่อหมวด และ JS ฟิลเตอร์ที่รอด View Transitions
- **KPI:** จำนวน session ที่ใช้ finder, คลิกจาก finder->หน้ารีวิว, landing จากคีย์ 'งบ X บาท'

#### [P3] ปุ่มโหวต 'ช่วยตัดสินใจได้/ยังสงสัย' ใต้ pick
- **คืออะไร:** ใต้แต่ละคำแนะนำใส่ปุ่มรีแอ็กชันเชิงบวกหลายแบบ (สไตล์ Pantip emotions) ให้ผู้อ่านบอกว่า pick ไหนช่วยตัดสินใจได้จริง แล้วใช้จัดอันดับ/เก็บ signal คุณภาพคำแนะนำ เริ่มเก็บ UGC ชั้นแรก
- **ต้นแบบ:** Pantip, RTINGS.com, Slickdeals
- **Friendsay ขาดอะไร:** current-state ระบุ 'ยังไม่มี UGC/รีวิวผู้ใช้จริงบนเว็บ' — ไม่มี feedback loop จากผู้อ่าน
- **effort:** med · **ติดที่:** ข้อมูล+เทคนิค — ต้องมี backend เก็บโหวต และ traffic มากพอจะ meaningful (cold-start)
- **KPI:** จำนวนโหวต/หน้า, สัดส่วนหน้าที่ได้โหวต, correlation โหวต vs conversion


## สิ่งที่ "ห้ามลอก" (พร้อมเหตุผล)

- **Affiliate link hijacking / cookie stuffing / แย่ง last-click แบบ Honey** — ทำลาย trust และผิดกฎหมาย (Honey โดน class action + Rakuten ตัด ~2,000 ร้าน ม.ค. 2026). ขัดจุดยืนความซื่อสัตย์ของ Friendsay โดยตรง — ให้ทำตรงข้าม: โปร่งใส ไม่ทับ attribution ใคร
- **ตัวนับ 'ซื้อเองมาทดสอบ X ชิ้น' / ห้องแลบจับของจริงแบบ RTINGS/Which?** — Friendsay ไม่มีสินค้าจริง (ข้อจำกัดหลัก) การโชว์ตัวเลขทดสอบ/hands-on จะเป็นการโกหกที่พังทันทีเมื่อถูกจับได้ (บทเรียน Honey 'อ้างเกินจริง'). ให้ชูความเป็นกลาง+หลักฐานราคา/สเปกย้อนได้แทน
- **Hard paywall / membership กั้นรีวิวแบบ Which?/RTINGS** — ฆ่า top-funnel SEO ซึ่งเป็นเส้นเลือดของโมเดล affiliate ไทย. Friendsay รายได้มาจาก traffic->คลิก affiliate การกั้นเนื้อหาจะตัดรายได้หลักทิ้ง
- **ขายอันดับ/ค่าธรรมเนียม license ป้าย 'Best Buy' ให้แบรนด์ (pay-to-rank)** — ขัด positioning ความเป็นกลางแบบตรงข้าม — จุดขายเดียวที่คู่แข่งไทยลอกไม่ได้คือ 'อันดับซื้อไม่ได้'. รับเงินจัดอันดับเมื่อไหร่ = กลายเป็น Mercular/สื่อรับสปอนเซอร์ที่ Friendsay กำลังจะชนะ
- **ดึงเรตติ้งรวมจาก marketplace มาโชว์แบบไม่ระบุแหล่ง (แบบ Priceza)** — เรตติ้งยืมมาปั่นได้และตรวจไม่ได้ ขัดกฎ B+D 'ไม่อ้างสิ่งที่พิสูจน์ไม่ได้'. ถ้าจะใช้เรตติ้ง ต้องระบุแหล่ง+จำนวนรีวิวเสมอ (ดูข้อ P1 chip เรตติ้ง)

## Global · รีวิวคัดสรร + ความเชื่อใจ


### Wirecutter (NYT) — nytimes.com/wirecutter
- **ประเภท:** บรรณาธิการฟันธง 1 คำตอบ (opinionated editorial single-pick)
- **โมเดล/ทำไมคนกลับมา:** เว็บรีวิวของ The New York Times ที่ให้ 'คำตอบเดียว' ต่อหมวดสินค้า รายได้หลักเดิมคือ affiliate (ปี 2024 ผลักดันยอดขาย referred ~120-150M USD, สะสมกำลังแตะ ~1 พันล้าน USD commerce ตาม Forbes 10/2025) เสริมด้วยโฆษณาที่เพิ่งเปิดปี 2024 (บรรลุเป้าโฆษณา 2025 ตั้งแต่กลางปี) และอยู่ใน 'Essential Bundle' ของ NYT (News+Games+Cooking+Wirecutter+The Athletic) ที่ครอบคลุมเกือบ 50% ของสมาชิก NYT 11.5 ล้านราย เนื้อหาอยู่หลัง metered paywall  
  _ตัดความเหนื่อยล้าในการเลือก (decision fatigue) ด้วยการฟันธงตัวเดียว + สายประหยัด อยู่ใต้แบรนด์ความน่าเชื่อ NYT และมี last-updated ที่รีเฟรชต่อเนื่อง ทำให้คนกลับมาเช็คว่า 'ตอนนี้อันไหนดีสุด' โดยไม่ต้องอ่านสเปกเอง_
- **กลไกลายเซ็น:**
  - **ลำดับชั้น pick ที่ตั้งชื่อชัด** — แต่ละบทความจัดเป็น Our pick / Runner-up / Budget pick / Upgrade pick / Also great (ตัวเลือกเฉพาะทาง) ผู้อ่านเลือกตามงบ/ความต้องการได้ทันทีโดยไม่ต้องเทียบเอง · *ทำไมได้ผล:* ลด decision fatigue — คนส่วนใหญ่ต้องการ 'satisficing' คือคำตอบที่ดีพอ ไม่ใช่ตารางสเปก 20 คอลัมน์ การตั้งชื่อ tier ทำให้ map ความต้องการ->สินค้า ได้ใน 3 วินาที
  - **กล่องความโปร่งใส 3 ส่วน** — ทุกบทความมี 'Why you should trust us' + 'How we picked' + 'How we tested' บอกชื่อผู้ทดสอบ (byline) ชั่วโมง/จำนวนที่ทดสอบ และเกณฑ์คัดเลือก · *ทำไมได้ผล:* แปลงความเห็นเชิงบรรณาธิการให้กลายเป็นกระบวนการที่ตรวจสอบได้ ลดข้อครหาว่าเชียร์เพราะค่าคอมมิชชัน — trust มาจากการ 'แสดงงาน' ไม่ใช่แค่ยืนยัน
  - **'Flaws but not dealbreakers'** — section ที่ระบุข้อเสียของตัวที่เลือกเองอย่างตรงไปตรงมา ก่อนสรุปว่าทำไมยังแนะนำ · *ทำไมได้ผล:* การยอมรับข้อเสียเชิงรุกสร้างความรู้สึกซื่อสัตย์ (costly signaling) คนเชื่อคำชมมากขึ้นเมื่อเห็นว่ากล้าติ
  - **แยกธุรกิจออกจากบรรณาธิการ** — ไม่มี sponsored content/paid placement — การคุยเรื่องลิงก์ affiliate เกิดขึ้น 'หลัง' ตัดสินใจ pick แล้วเท่านั้น ประกาศจุดยืนนี้ชัดในหน้า about · *ทำไมได้ผล:* ตัดแรงจูงใจที่บิดเบือนคำแนะนำ ทำให้ผู้อ่านเชื่อว่า pick มาจากคุณภาพ ไม่ใช่ค่าคอมสูงสุด
- **Trust device:** Byline ผู้ทดสอบมีตัวตน + เครดิต NYT; ประกาศ 'ไม่รับ sponsored/paid placement' ชัดเจน; ระบุจำนวน/ชั่วโมงที่ทดสอบต่อบทความ; Last updated date ที่รีเฟรชบ่อย; หน้าถาวร How We Test อธิบายกระบวนการรวม
- **รายได้:** Affiliate commission (แกนหลัก) + โฆษณา (เริ่ม 2024) + ส่วนแบ่งจาก NYT subscription bundle หลัง metered paywall — โมเดลผสม affiliate+membership
- **จุดอ่อน/ความเสี่ยง:** pick มักถูกวิจารณ์ว่า 'satisficing/generic' — ดีพอแต่ไม่ลึกสำหรับ enthusiast; พึ่ง Amazon/affiliate สหรัฐเป็นหลัก สินค้า/ลิงก์ไม่ตรงตลาดอื่น; paywall สร้าง friction กับ top-of-funnel SEO; scale กว้างทำให้ความลึกการทดสอบต่อชิ้นสู้ห้องแลบเฉพาะทาง (RTINGS) ไม่ได้
- **ปรับมาใช้กับ Friendsay:**
  - template lพick tier ตั้งชื่อชัดทุกบทความ → กำหนด component ตายตัว: 'ตัวเลือกหลัก / สายประหยัด / รุ่นอัปเกรด / ตัวเลือกเฉพาะทาง' วางบนสุดก่อนเนื้อหายาว พร้อมปุ่มซื้อ affiliate ต่อ tier (effort low, ติดที่: เจ้าของต้องล็อก template กลาง + ทำเป็น Astro component ที่รอด client-nav (View Transitions))
  - กล่อง 'ทำไมต้องเชื่อเรา / เราคัดยังไง' ต้นบทความ → กล่องสรุป 3-4 บรรทัด: ใครรีวิว, ทดสอบกี่รุ่น/กี่ชั่วโมง, เกณฑ์คัด, วันอัปเดตล่าสุด — reusable snippet (effort low, ติดที่: ต้องมีตัวตน/ชื่อผู้รีวิวจริงและข้อมูลการทดสอบจริง (ติดเจ้าของ))
  - section 'ข้อเสียที่ไม่ใช่ตัวตัดสิน' → บังคับทุกบทความมีหัวข้อระบุจุดด้อยของตัวที่เชียร์ 2-3 ข้อ ก่อนสรุปเหตุผลที่ยังแนะนำ (effort low, ติดที่: ไม่มี — เป็นเรื่องแนวทางเขียน (editorial guideline))

### RTINGS.com
- **ประเภท:** ห้องแลบวัดผลเชิงตัวเลข (quantified in-house lab benchmarks)
- **โมเดล/ทำไมคนกลับมา:** เว็บทดสอบอิสระ 100% ที่ 'ซื้อทุกเครื่องเอง' เพื่อกันการ cherry-pick ตัวอย่างจากแบรนด์ หน้าแรกโชว์ตัวเลขความน่าเชื่อชัด: '4,734 Bought & Tested', '40,000 Sq.Ft. Testing Facility' (มีวิดีโอทัวร์), '28 Product Categories', '100% Independent' รายได้จาก affiliate + membership ที่ปลดล็อกผลทดสอบเต็ม/เครื่องมือเทียบ  
  _เป็นแหล่ง 'ตัวเลขกลาง' ที่ enthusiast อ้างอิงได้ — คนกลับมาใช้ Compare tool และ Results Table เทียบรุ่นต่อรุ่นด้วยคะแนนวัดจริง และเช็คคะแนนที่อัปเดตตาม test bench เวอร์ชันใหม่_

  - **Test bench แบบมีเวอร์ชัน + คะแนนถ่วงตาม use-case** — เผยแพร่ methodology + รูป + วิดีโอของทุกเทสต์ ที่หน้า /company/test-benches-and-scoring-system คะแนนเป็นแบบ weighted 'Mixed Usage' และปรับตามการใช้งาน (เกมเน้น input lag/VRR, หนังเน้น contrast/สี) มี test bench เวอร์ชัน (overhaul 2021, RTINGS 2.0) · *ทำไมได้ผล:* การ 'publish the methods, photos, and videos' ทำให้คะแนนตรวจสอบซ้ำได้ = วิทยาศาสตร์ ไม่ใช่ความเห็น; versioning เป็น changelog สาธารณะที่บอกว่าเกณฑ์ทันสมัย

## Global · เทียบเชิงโปรแกรม + ครองหมวด


### versus.com
- **ประเภท:** เครื่องเทียบเชิงโปรแกรมข้ามหมวด (programmatic X-vs-Y engine) + คะแนนกลางที่ซื้อไม่ได้
- **โมเดล/ทำไมคนกลับมา:** สร้างหน้าเทียบอัตโนมัติจากฐานสเปกกลาง: ผู้ใช้พิมพ์สองสิ่งที่จะเทียบ ระบบดึงสเปกเทียบทีละแอตทริบิวต์แล้วออกหน้า X vs Y ครอบคลุมกว่า 26 ล้านคู่เปรียบเทียบ, 100+ หมวด, 15 ภาษา. รายได้จาก affiliate/โฆษณา แต่ประกาศชัดว่าคะแนนซื้อด้วยเงินไม่ได้  
  _ตอบคำถามเดียวที่คนค้างคาใจตอนจะซื้อ ('ตัวไหนดีกว่ากัน') ด้วยหน้าเดียวจบ ปรับน้ำหนักตามสิ่งที่ตัวเองแคร์ได้ และมีหน้าเทียบแทบทุกคู่ที่นึกออกจากการค้นหา จึงมักโผล่บน Google เป็นคำตอบสำเร็จรูป_
- **กลไกลายเซ็น:**
  - **ป้ายนับ 'facts in comparison'** — หัวหน้าเทียบโชว์ตัวนับจำนวนข้อเท็จจริงที่นำมาเทียบจริงในคู่นั้น (เจ้าของยกตัวอย่าง '257 FACTS IN COMPARISON') ตัวเลขคำนวณจากจำนวนสเปกที่แมตช์กันในคู่นั้นจริง ๆ · *ทำไมได้ผล:* แปลง 'ความละเอียด' ให้เป็นตัวเลขเดียวที่ประเมินความน่าเชื่อถือได้ทันที ทำให้ผู้ใช้รู้สึกว่านี่คือการเทียบที่ครอบคลุมกว่าบล็อกรีวิวทั่วไป โดยไม่ต้องอ่านทั้งหน้า
  - **บล็อก 'Why is X better than Y?' เป็นข้อเท็จจริงเชิงส่วนต่าง** — ลิสต์เหตุผลแบบ bullet ที่แต่ละข้อเป็นค่าต่างแบบวัดได้ (เช่น จอใหญ่กว่า 6.7" vs 6.1", แบตมากกว่า, ชาร์จไวกว่า) แยกสองฝั่งว่าทำไม X ชนะและทำไม Y ชนะ · *ทำไมได้ผล:* เปลี่ยนตารางสเปกดิบให้เป็นคำตัดสินที่อ่านแล้วเข้าใจทันที ลดภาระคิดของผู้ใช้ และเป็นถ้อยคำที่ Google หยิบไปทำ snippet ได้ตรง ๆ ตามคีย์เวิร์ด 'X vs Y'
  - **Versus score ที่ถ่วงน้ำหนักด้วยโหวตของผู้ใช้ + Wilson score** — ทุกสินค้าเริ่มจากสเปกเทียบกับค่าเฉลี่ยในหมวด รองรับกรณี 'น้อยกว่าดีกว่า' (เบากว่า/กินไฟน้อยกว่าได้คะแนนดีกว่า) มีปุ่มโหวตข้างทุกสเปกให้ผู้ใช้บอกว่าอะไรสำคัญกว่า รีวิวผู้ใช้มีผลต่อคะแนนได้ถึง 20% โดยใช้ Wilson score confidence interval กันไม่ให้ของที่มีรีวิวเดียวคะแนนเต็มแซงของที่มีรีวิวดีเป็นพัน รันทดสอบใหม่ทุก 3 เดือน · *ทำไมได้ผล:* ทำให้คะแนนดูเป็นวิทยาศาสตร์และป้องกันการปั่น เป็นตัวเลขกลางที่อ้างอิงได้ในทุกหน้า สร้างความรู้สึก 'เป็นกลาง' ที่แตกต่างจากรีวิวเชียร์ขาย
  - **ตัวปรับลำดับความสำคัญ (reprioritize) ต่อผู้ใช้** — ผู้ใช้เลือกได้ว่าให้แอตทริบิวต์ไหนสำคัญกว่า (เช่นแบต > จอ) แล้วผลเทียบ/คะแนนปรับตาม · *ทำไมได้ผล:* ยอมรับว่า 'ดีที่สุด' ขึ้นกับความต้องการแต่ละคน ทำให้คำแนะนำรู้สึกเป็นของฉัน เพิ่ม engagement และเวลาอยู่บนหน้า
  - **ตาข่ายลิงก์ภายในจาก search + หน้าเทียบยอดนิยม** — ทุกสินค้ามีปุ่มเทียบด่วน, กล่อง 'popular comparisons' และหน้าหมวดที่กรอง/เรียงตามสเปก (ISO, ความละเอียด, ราคา) ที่โยงเข้าหน้าเทียบนับล้านหน้า · *ทำไมได้ผล:* สร้าง long-tail SEO มหาศาลแบบ programmatic ครอบคลุมทุกคู่ที่คนจะค้น และเก็บผู้ใช้ให้ไหลจากคู่หนึ่งไปอีกคู่โดยไม่ออกจากเว็บ
- **Trust device:** ประกาศชัด 'คะแนนซื้อด้วยเงินไม่ได้' และหน้า Editorial Guidelines; ระบุแหล่งข้อมูล: เว็บผู้ผลิต + คู่มือ + การทดสอบจริง; ป้ายนับจำนวนข้อเท็จจริงและตัวเลขคู่เปรียบเทียบ 26M+ เป็นสัญญาณสเกล; อธิบายสูตรคะแนน (Wilson score, รีทดสอบทุก 3 เดือน) แบบโปร่งใสในบทความ 'hidden complexity of Versus score'
- **รายได้:** Affiliate ไปยังร้านค้า/มาร์เก็ตเพลส + โฆษณาดิสเพลย์ ปริมาณมหาศาลจาก long-tail SEO ชดเชย conversion ต่อหน้าที่ต่ำ

- **ประเภท:** Amazon-only price-history tracker + price-drop alert service (free, consumer-first)
- **โมเดล/ทำไมคนกลับมา:** ผูกกับ Amazon ล้วน ๆ: ทุกหน้าสินค้ามีกราฟราคาย้อนหลัง 3 เส้น (Amazon = เขียว, 3rd-Party New = น้ำเงิน, 3rd-Party Used = แดง) พร้อม high/low/average และช่วงเวลา 1m/3m/6m/1y/all คนตั้ง 'Price Watch' ใส่อีเมล + ราคาที่อยากได้ เมื่อราคาตกถึงจุดนั้นจะส่งอีเมลเตือน ฟรีทั้งหมด รายได้มาจาก Amazon affiliate เมื่อคนกดลิงก์ไปซื้อ  
  _ก่อนกดซื้อของ Amazon ทุกครั้ง คนเช็คว่าราคาปัจจุบัน 'ดีจริงหรือแค่ลดหลอก' เทียบกับ low/average ในอดีต — เป็นพฤติกรรม gatekeeper ก่อนจ่ายเงิน และอีเมลเตือนดึงคนกลับมาเองเมื่อราคาถึงเป้า_
- **กลไกลายเซ็น:**
  - **กราฟราคา 3 เส้น + high/low/average** — แยกเส้น Amazon / 3rd-Party New / Used ด้วยสี พร้อมแถบ 'ราคาต่ำสุด-สูงสุด-เฉลี่ย' ตายตัวข้างกราฟ และสไลด์ช่วงเวลาได้ถึง 'all' (ตั้งแต่วันแรกที่ลงขาย) · *ทำไมได้ผล:* เปลี่ยน 'ราคาถูก' ที่เป็นความรู้สึกให้เป็นข้อเท็จจริงเชิงตัวเลข คนเห็นทันทีว่าดีลนี้อยู่ตรงไหนของประวัติ ลดความกลัวโดนหลอกลดราคา
  - **Price Watch (ตั้งเตือนราคาด้วยอีเมล)** — ช่องตั้งเตือนอยู่ 'เหนือกราฟ' ทุกหน้า ใส่อีเมล + ราคาเป้าหมาย ระบบเก็บ watch ไว้แล้วยิงอีเมลเมื่อถึงเป้า ไม่ต้องล็อกอิน · *ทำไมได้ผล:* จับ demand ที่ยังไม่พร้อมซื้อ (รอราคาลง) แล้วดึงกลับมาเองผ่านอีเมล = retention loop ที่ไม่ต้องมี app
  - **Camelizer browser extension** — ปลั๊กอิน Chrome/Edge/Firefox/Opera/Safari โชว์กราฟราคาซ้อนบนหน้า Amazon จริง และตั้ง watch ได้ในคลิกเดียว รองรับ Amazon US/CA/UK/DE/ES/FR/IT/AU + JP/CN · *ทำไมได้ผล:* เอาข้อมูลไปวางตรงจุดตัดสินใจ (บนหน้าสินค้าเอง) ไม่ต้องเปิดเว็บแยก ลด friction จนเป็นนิสัย
  - **Wishlist Importer** — วางลิงก์ Amazon public list แล้วระบบ import สินค้าที่ยังไม่ได้ซื้อ มาตั้ง watch อัตโนมัติที่ 'Amazon new price ลบ 1' · *ทำไมได้ผล:* เปลี่ยน wishlist ที่นิ่ง ๆ ให้กลายเป็นเครื่องยิงดีลอัตโนมัติ = สร้าง watch จำนวนมากจากคนเดียวในคลิกเดียว
- **Trust device:** ฟรีทั้งหมด ไม่บังคับล็อกอินเพื่อตั้ง watch; แสดง high/low/average เป็นตัวเลขที่ตรวจย้อนได้ ไม่ใช่คำพูดลอย; อยู่มานาน (แบรนด์ที่ reviewer/พ่อค้าใช้อ้างอิงกันเอง); โปร่งใสว่าเป็น Amazon affiliate
- **รายได้:** Amazon Associates affiliate — คนกดจากกราฟ/หน้าไปซื้อบน Amazon เว็บได้ค่าคอมมิชชัน (ไม่คิดเงินผู้ใช้เลย)
- **จุดอ่อน/ความเสี่ยง:** Amazon-only — ไม่เทียบข้ามร้าน ใช้ไม่ได้กับตลาดที่ Amazon ไม่ใช่เจ้าหลัก (เช่นไทย); เตือนผ่านอีเมลเท่านั้น ไม่ push เรียลไทม์ตอนอยู่หน้าจ่ายเงิน อาจช้ากว่าราคาจริง; ข้อมูลบางช่วง lag/ขาด ต้องพึ่ง Amazon API ที่บริษัทควบคุมไม่ได้; พึ่งพารายได้จาก Amazon ล้วน ๆ เสี่ยงถ้า affiliate term เปลี่ยน
- **ปรับมาใช้กับ Friendsay:**
  - แถบ 'ราคานี้ดี/แพงเทียบ 90 วัน' ตายตัวบนทุกหน้ารีวิว/สินค้า → เก็บราคาจากร้านไทย (Shopee/Lazada/JD) รายวัน แล้วโชว์ badge เช่น 'ต่ำสุดใน 90 วัน' / 'สูงกว่าเฉลี่ย 12%' + กราฟย่อ sparkline ใต้ปุ่มซื้อ (effort high, ติดที่: ต้องมี pipeline เก็บราคาย้อนหลัง (scrape/API ร้านไทย) + storage — ติดข้อมูล ไม่ติดเจ้าของ)
  - Price Watch แบบไม่ต้องล็อกอิน (ใส่อีเมล/LINE + ราคาเป้า) → ช่อง 'แจ้งเตือนเมื่อราคาถึง ___ บาท' เหนือปุ่มซื้อ ส่งผ่าน LINE Notify/อีเมล ดึงกลับมาซื้อผ่านลิงก์ affiliate ของเรา (effort med, ติดที่: ต้องมีข้อมูลราค่าเรียลไทม์ + ตัวส่ง LINE/อีเมล; ต้องรอด client-nav ของ View Transitions)
  - 'ต่ำสุดในรอบปี' เป็น hook ในหัวข้อ roundup → ในบทเทียบสินค้ายอดฮิต ใส่ป้าย 'ตอนนี้ถูกสุดในรอบ 6 เดือน' ข้างตัวที่แนะนำ อัปเดตอัตโนมัติ เพิ่มความเร่งด่วนให้กดซื้อ (effort low, ติดที่: ถ้ายังไม่มี price history จริง ใช้แบบ manual/รายสัปดาห์ไปก่อนได้)

### Keepa (keepa.com)
- **ประเภท:** Amazon price-history + seller-data platform (freemium + API), เชิงลึกสุดในคลัสเตอร์
- **โมเดล/ทำไมคนกลับมา:** ปลั๊กอิน/เว็บที่ฝังกราฟราคาละเอียดบนหน้า Amazon จริง: New/Used/Warehouse Deals/Buy Box/Sales Rank/Offer count ซูมได้ตั้งแต่รายวันถึงหลายปี แท็บ 'Track' ตั้งราคาที่อยากได้ + % ที่ต้องลด เตือนผ่านอีเมล/แอป/Discord/Telegram/browser push มี Deals feed, Product Finder, Seller Lookup รายได้จาก Pro subscription (~€19-29/เดือน มี data token) + ขาย API ให้ dev (ธุรกิจดั้งเดิม) + affiliate  
  _นักขาย Amazon (arbitrage/FBA) และนักช้อปจริงจังใช้ Keepa เป็นเครื่องมือ 'ตัดสินใจซื้อ/สต็อก' ทุกวัน เพราะเป็น source of truth เรื่องราคา+ยอดขายที่ละเอียดที่สุด กราฟฝังบนทุกหน้า Amazon จนกลายเป็นส่วนหนึ่งของ workflow_
- **กลไกลายเซ็น:**
  - **กราฟ multi-layer ฝังบนหน้า Amazon** — เส้นราคาหลายชั้น (Amazon/New/Used/Warehouse/Buy Box) + Sales Rank + จำนวน offer ในกราฟเดียว ซูม day↔years เปิด/ปิดแต่ละเส้นได้ · *ทำไมได้ผล:* ให้ 'ความจริงหลายมิติ' ในที่เดียว — ไม่ใช่แค่ราคาถูก แต่ราคาถูก+ของขายดี+มีคนแข่งเยอะไหม ทำให้ตัดสินใจได้ลึกกว่าคู่แข่ง
  - **Track: desired price + drop %** — ตั้งได้ทั้ง 'ราคาบาท/ดอลลาร์ที่ต้องการ' และ '% ที่ต้องลดจากปัจจุบัน' ต่อเงื่อนไข New/Used แยกกัน แจ้งหลายช่องทาง (email/app/Discord/Telegram/push) · *ทำไมได้ผล:* ยืดหยุ่นกว่าตั้งราคาตายตัว (% เหมาะกับของที่ราคาแกว่ง) + multi-channel ทำให้เตือนถึงเร็วตอนราคาลง = ปิดการขายได้ทัน
  - **Deals feed + Product Finder** — หน้ารวมสินค้าที่เพิ่งลดราคา กรองด้วย % ลด, หมวด, rating ขั้นต่ำ, ช่วงราคา; Product Finder ค้นสินค้าตามเกณฑ์ sales rank/ราคา · *ทำไมได้ผล:* เปลี่ยนจาก 'รอเตือนทีละชิ้น' เป็น 'เดินดูดีลทั้งตลาด' — สร้าง session ยาวและ discovery ที่นำไปสู่การซื้อ/สต็อก
  - **โมเดล data-token / API** — Pro ให้โควตา token สำหรับดึงข้อมูลเชิงลึก, ขาย API ให้ผู้พัฒนา (ธุรกิจดั้งเดิมก่อนมี consumer plan) · *ทำไมได้ผล:* monetize 'ข้อมูลราคา' โดยตรง ไม่พึ่ง affiliate อย่างเดียว = โมเดลรายได้ทนทาน และ B2B/dev กลายเป็น moat ของข้อมูล
- **Trust device:** Free-forever plan ไม่ต้องกรอกบัตร; ข้อมูลละเอียดตรวจย้อนได้หลายปี = เชื่อถือได้เชิงเทคนิค; ถูกใช้เป็นมาตรฐานในคอมมูนิตี้นักขาย Amazon; รองรับ 11+ marketplace (US/DE/UK/FR/IT/ES/CA/JP/IN/MX/BR)
- **รายได้:** Freemium subscription (Pro ~€19-29/เดือน, มี data token), ขาย API ให้ developer, และ Amazon affiliate
- **จุดอ่อน/ความเสี่ยง:** UI ซับซ้อน เอนไปทางนักขาย/มือโปร ไม่เป็นมิตรกับนักช้อปทั่วไป; Amazon-only — ไม่เทียบข้ามร้านนอก Amazon; ฟีเจอร์ลึกอยู่หลัง paywall + ระบบ token ทำให้คนใหม่งง; พึ่งข้อมูล Amazon เป็นหลัก เสี่ยงถ้า Amazon จำกัดการเข้าถึง
- **ปรับมาใช้กับ Friendsay:**
  - แจ้งเตือนแบบ 'ลดกี่ %' ไม่ใช่แค่ราคาตายตัว → ในระบบเทียบสินค้ายอดฮิต ให้ผู้ใช้เลือก 'เตือนเมื่อลด ≥15% จากราคาเฉลี่ย' นอกจากตั้งบาทเป๊ะ ๆ ส่งผ่าน LINE (effort med, ติดที่: ต้องมีราคาเฉลี่ยย้อนหลังเก็บไว้ก่อน — ติดข้อมูล)
  - หน้า Deals feed ของสินค้าไทยที่ 'เพิ่งลดจริง' → หน้า /deals รวมสินค้าในหมวด catalog 6 หมวดที่ราคาตกต่ำกว่าเฉลี่ย 30 วัน กรองด้วย % ลด/หมวด/ช่วงราคา ลิงก์ออก affiliate (effort high, ติดที่: ต้องมี price pipeline + cron อัปเดต; ต้องรอด View Transitions client-nav)
  - แสดง 'สัญญาณหลายมิติ' ไม่ใช่แค่ราคา → ข้างสินค้าแนะนำ โชว์ป้ายเสริม เช่น 'ขายดีอันดับต้นในหมวด' + 'ราคาต่ำกว่าเฉลี่ย' ควบคู่กัน เพิ่มน้ำหนักการตัดสินใจ (effort med, ติดที่: ต้องดึง ranking/ยอดขายจากร้าน — ติดข้อมูล (บางร้านไม่เปิด))

### idealo (idealo.de, EU)
- **ประเภท:** Cross-shop price-comparison marketplace + price history/alert (เยอรมนี/EU เจ้าตลาด)
- **โมเดล/ทำไมคนกลับมา:** เทียบราคาสินค้าเดียวกันข้าม 'ร้านหลายพันร้าน': 183 ล้าน offer จาก ~30,000 ร้าน ทุกหน้าสินค้ามี Preisverlauf (กราฟราคา วัน/สัปดาห์/เดือน) + Preiswecker/Preisalarm (ตั้งราคาที่ต้องการแล้วแจ้งเตือน) ค้นด้วยข้อความ/EAN/สแกนบาร์โค้ด มีเทสต์รีพอร์ต รีวิว วิดีโอ รายได้จาก CPC — เก็บเงินร้านค้าต่อคลิกที่ส่งออกไป (มาตรฐาน ~€0.51/คลิก, UK £0.24, ขั้นต่ำ €20/เดือน) + มีขายตรง (idealo Direktkauf)  
  _ผู้บริโภค EU เช็ค idealo ก่อนซื้อของออนไลน์แทบทุกครั้งเพื่อหา 'ร้านไหนถูกสุด + ราคานี้ใช่จังหวะไหม' — เป็น default price checker ข้ามร้าน ไม่ผูกกับร้านเดียว จึงเป็นกลาง_
- **กลไกลายเซ็น:**
  - **เทียบราคาข้ามร้านหลายพันร้าน** — หน้าสินค้าเดียวลิสต์ราคาจากทุกร้านเรียงถูก→แพง พร้อมค่าส่ง; ครอบคลุม 183M offer / ~30,000 ร้าน · *ทำไมได้ผล:* ตอบคำถาม 'ร้านไหนถูกสุด' ในหน้าเดียว = คุณค่าหลักของ comparison ที่ tracker Amazon-only ให้ไม่ได้ ทำให้เป็นด่านก่อนซื้อ
  - **Preisverlauf (กราฟราคาย้อนหลัง)** — โชว์การเคลื่อนของราคา วัน/สัปดาห์/เดือน ให้ดูว่า 'ตอนนี้ถูกที่สุดในรอบนานหรือกำลังขึ้น' · *ทำไมได้ผล:* รวม 'ถูกสุดข้ามร้าน' (มิติพื้นที่) กับ 'ถูกสุดข้ามเวลา' (มิติเวลา) — ตัดสินใจได้ครบทั้งจังหวะและร้าน
  - **Preiswecker/Preisalarm** — ตั้งราคาที่อยากได้ ระบบแจ้งเตือนเมื่อร้านใดร้านหนึ่งลงถึงเป้า · *ทำไมได้ผล:* จับ demand ที่ยังไม่พร้อมจ่าย แล้วดึงกลับมาเมื่อราคาถึง = retention + ปิดการขายผ่านคลิก CPC ของ idealo
  - **ค้นด้วย EAN/บาร์โค้ด + เทสต์รีพอร์ต** — สแกนบาร์โค้ดในแอปเพื่อเทียบราคาสินค้าเดียวกันทันที + รวม Testberichte/รีวิว/วิดีโอ/ดาต้าชีต · *ทำไมได้ผล:* ทำให้เทียบราคาได้แม้ยืนอยู่ในร้านจริง (offline→online) และเสริมความมั่นใจด้วยรีวิว/ผลทดสอบก่อนกดซื้อ
- **Trust device:** ตัวเลขสเกลชัด: 183M offer / ~30,000 ร้าน; เป็นกลาง — ไม่ผูกกับร้านเดียว จึงดูน่าเชื่อ; รวมเทสต์รีพอร์ต/รีวิวอิสระ + ดาต้าชีต; โมเดล CPC โปร่งใส ไม่มีค่าธรรมเนียมแฝงฝั่งร้าน
- **รายได้:** CPC (pay-per-click) — เก็บเงินร้านค้าต่อคลิก out ~€0.51 มาตรฐาน (UK £0.24), ขั้นต่ำ €20/เดือน; เสริมด้วยขายตรง idealo Direktkauf และ retail media
- **จุดอ่อน/ความเสี่ยง:** โฟกัส DE/EU — coverage และ UX นอกยุโรปจำกัด; CPC bias: ร้านที่ยอมจ่าย/bid อาจถูกดันขึ้น อาจไม่ใช่ 'ถูกสุดจริง' เสมอ; ราคาอาจ lag กับหน้าจ่ายจริงของร้าน (ค่าส่ง/โปรโค้ดไม่ตรง); ร้านเล็กหรือ niche อาจไม่อยู่ในระบบ ทำให้เทียบไม่ครบ
- **ปรับมาใช้กับ Friendsay:**

  - รวมป้าย 'ถูกสุดข้ามร้าน' + 'ถูกสุดข้ามเวลา' เข้าด้วยกัน → badge คู่: 'ร้าน X ถูกสุดตอนนี้' และ 'ราคานี้ต่ำสุดในรอบ 3 เดือน' บนการ์ดสินค้าแนะนำในบท roundup (effort med, ติดที่: ต้องเก็บราคาหลายร้าน + history; เริ่มจาก 2-3 ร้านหลักก่อนได้)

## Global · ชุมชนดีล + กรณีศึกษา "ห้ามลอก"


### Slickdeals (slickdeals.net)
- **ประเภท:** ชุมชนดีล crowdsourced + curation ลำดับชั้น (ชุมชนโพสต์/โหวต -> บก.กลั่นกรอง -> Front Page) พร้อมระบบแจ้งเตือนดีลรายบุคคล

- **กลไกลายเซ็น:**
  - **Deal thermometer / โหวตชุมชน (thumbs up-down -> Hot/Good/Okay)** — ทุกดีลโชว์จำนวนโหวตบวก/ลบแบบเปิดเผย ระบบจัดอันดับตามความนิยมแล้วติดป้าย 'hot' เมื่อได้โหวตบวกเยอะ (เช่นบวก 500 ลบ 20) ดันดีลห่วยลงล่างอัตโนมัติ · *ทำไมได้ผล:* แปลง 'wisdom of the crowd' เป็นสัญญาณเดียวที่อ่านง่ายในเสี้ยววินาที ลดภาระตัดสินใจ + เป็น social proof สด ๆ ที่ปลอมยากเพราะโหวตมาจากคนจริงจำนวนมาก
  - **Curation สองชั้น: Popular Deals -> บก.กลั่น -> Front Page** — ดีลต้องผ่านโหวตชุมชนขึ้นเป็น Popular ก่อน จากนั้น Deal Editors คนจริงคัดอีกชั้นว่าดีลไหนคู่ควร Front Page จุดขายคือ 'Front Page is earned, not bought' · *ทำไมได้ผล:* รวมข้อดีของ crowd (สเกล, จับดีลไว) กับ editorial (กันสแปม/ดีลปลอม) ป้ายว่าซื้อไม่ได้สร้าง trust ว่าอันดับไม่ถูกเงินซื้อ ต่างจากเว็บ affiliate ทั่วไปที่จ่ายแล้วขึ้นก่อน

  - **Reputation points ให้คนโพสต์** — ผู้ใช้มีคะแนน reputation (บวก/ลบ/กลาง) ตามคุณภาพโพสต์ คนโพสต์ดีลดังบ่อยได้เครดิตในชุมชน · *ทำไมได้ผล:* gamification ที่ทำให้ 'ผู้จัดหาคอนเทนต์ฟรี' ทำงานให้ต่อเนื่องโดยไม่ต้องจ่ายเงิน สร้าง flywheel ของ UGC ที่คู่แข่งลอก layout ได้แต่ลอกชุมชนไม่ได้
  - **พิธีกรรมดีลตามปฏิทิน (Black Friday / Prime Day hub + เมกะเธรด)** — ช่วงเทศกาลช้อปมีหน้า hub รวมดีลอัปเดตสด + เธรดฟอรัมยาวที่ชุมชนแชร์เรียลไทม์ กลายเป็นจุดนัดพบประจำปี (อ้างจากความรู้ทั่วไป + โครงสร้างฟอรัม SD, ไม่ได้เปิดหน้าเฉพาะกิจ ณ ตอนวิเคราะห์) · *ทำไมได้ผล:* สร้าง 'นัดหมาย' ที่คาดเดาได้ ดึงคนกลับพร้อมกันเป็นคลื่น เพิ่ม engagement/ทราฟฟิกพีคและตอกย้ำนิสัยเช็ค SD ทุกครั้งที่จะช้อปใหญ่

### Honey / PayPal (joinhoney.com) — กรณีศึกษา 'ห้ามลอก'
- **ประเภท:** Anti-pattern เตือนใจ: coupon/rewards extension ที่ทำลายความไว้ใจตัวเองด้วยการแย่ง affiliate attribution + ซ่อนโค้ดที่ดีกว่า
- **โมเดล/ทำไมคนกลับมา:** เดิม: browser extension auto-apply โค้ดส่วนลดตอน checkout + Honey Gold rewards ฐานผู้ใช้ใหญ่ (PayPal ซื้อ ~4 พันล้าน USD ปี 2020) รายได้จาก affiliate commission แบบ last-click แต่ได้มาโดย 'ปล้อน' attribution ของครีเอเตอร์ที่ส่งลูกค้ามาจริง  
  _เดิมกลับมาเพราะความสะดวก auto-apply โค้ดตอนจ่ายเงิน 'โดยไม่ต้องหาเอง' — แต่หลัง MegaLag แฉ (ธ.ค. 2024) trust พังทั้งกับผู้ใช้และครีเอเตอร์ ม.ค. 2026 Rakuten Advertising ตัด Honey ออกจากเครือข่าย ~2,000 ร้าน (Walmart, Sephora, Lego, Dyson, Uniqlo)_
- **กลไกลายเซ็น:**
  - **Affiliate link hijacking / cookie stuffing ตอน checkout** — Honey เด้ง popup ตอนจบ checkout แม้ไม่ได้ใช้โค้ดเลย ถ้าผู้ใช้แตะอะไรก็ตาม มันเขียนทับ affiliate link เป็นของ Honey = attribution การขายไปเป็นของ Honey แทนครีเอเตอร์ที่ส่งลูกค้ามา · *ทำไมได้ผล:* (ในแง่รายได้ระยะสั้น) ดูดค่าคอมจาก last-click ได้มหาศาลโดยไม่ต้องสร้างคอนเทนต์เอง — แต่คือการปล้นพันธมิตรที่หล่อเลี้ยง ecosystem ตัวเอง จุดชนวนคดี class action (In re PayPal Honey, N.D. Cal, 25+ คดี, ครีเอเตอร์ LegalEagle ร่วมฟ้อง)
  - **กั๊กโค้ดที่ดีกว่า / อ้าง 'ดีที่สุด' ทั้งที่ไม่จริง** — โฆษณาว่า 'กวาดทั้งเน็ตหาโค้ดที่ดีที่สุดให้คุณ' แต่จริง ๆ ให้ร้านค้าควบคุมได้ว่าจะโชว์โค้ดไหน มักโชว์โค้ดส่วนลดต่ำกว่าที่มีอยู่จริง · *ทำไมได้ผล:* (ในแง่ดีลกับเมอร์ชานต์) ร้านจ่ายเพื่อคุมส่วนลด = รายได้ B2B — แต่ทำลายคุณค่าหลักที่สัญญากับผู้ใช้ พอถูกจับได้ว่าโกหก คุณค่าทั้งแบรนด์พังทันที
- **Trust device:** เดิมพึ่งคำโฆษณา 'หาโค้ดที่ดีที่สุดให้อัตโนมัติ' + แบรนด์ PayPal หนุนหลัง — ทั้งคู่กลับกลายเป็นดาบสองคมเพราะคำอ้างพิสูจน์แล้วว่าเกินจริง = trust ติดลบหนักกว่าเว็บที่ไม่เคยอ้าง
- **รายได้:** Affiliate commission แบบ last-click (ได้มาโดย hijack attribution) + ดีล B2B กับเมอร์ชานต์ที่จ่ายเพื่อคุมโค้ด — ทั้งสองทางขัดผลประโยชน์ผู้ใช้/พันธมิตร จึงระเบิดเมื่อถูกเปิดโปง
- **จุดอ่อน/ความเสี่ยง:** แย่ง attribution ครีเอเตอร์ = ทำลายมือที่ป้อนลูกค้าให้ตัวเอง เมื่อครีเอเตอร์แฉพร้อมกัน แบรนด์ตายเร็ว (viral distrust); อ้างเกินจริง 'ดีที่สุด' ในสิ่งที่ตรวจสอบได้ = ระเบิดเวลา วันที่ผู้ใช้จับได้คือวันที่คุณค่าหลักเป็นศูนย์; ผลประโยชน์ทับซ้อน (รับเงินร้านคุมโค้ด) ขัดกับสัญญาที่ให้ผู้ใช้ ยั่งยืนไม่ได้; เครือข่าย affiliate ลงโทษได้จริง: Rakuten ตัดขาด ~2,000 ร้าน = รายได้หายทันทีเมื่อความน่าเชื่อถือหมด
- **ปรับมาใช้กับ Friendsay:**
  - ทำตรงข้าม Honey: โปร่งใสเรื่อง affiliate + ไม่แย่ง last-click ของใคร → เปิดเผยชัดทุกลิงก์ว่าเว็บได้ค่าคอม + ไม่ใช้ redirect/cookie ที่ทับ attribution คนอื่น ใส่หน้า 'เราหาเงินยังไง' ที่อธิบายตรงไปตรงมา (effort low, ติดที่: ไม่ติดข้อมูล/เทคนิค ติดแค่การตัดสินใจของเจ้าของว่ายอมเปิดเผยและไม่เล่นเกมสีเทา)
  - อย่าอ้าง 'ถูกที่สุด/ดีที่สุด' ถ้าพิสูจน์ไม่ได้ — โชว์วิธีหา + timestamp ราคา → แทนคำว่า 'ดีที่สุด' ด้วย 'ราคาที่เราเจอ ณ วัน/เวลา X จากร้าน A,B,C' พร้อมลิงก์ให้ผู้ใช้เช็คเอง — คำอ้างที่ตรวจสอบได้แทนคำอ้างที่ระเบิดได้ (effort low, ติดที่: ต้องมีกระบวนการ verify + บันทึก timestamp ราคาให้สม่ำเสมอ (ผูกกับระบบ Deal Alert/price feed ได้))
  - ให้ดีล/โค้ดที่ดีที่สุดจริง ไม่กั๊กเพื่อแลกดีลกับแบรนด์ → นโยบายชัดเจนว่าจะไม่รับเงินแลกกับการซ่อนส่วนลดที่ดีกว่า ถ้ามีโค้ดดีกว่าจากที่อื่นก็บอก แม้ค่าคอมน้อยกว่า — สร้าง trust ระยะยาวแทนกำไรระยะสั้น (effort med, ติดที่: แรงจูงใจขัดกัน (ดีลค่าคอมสูงvsดีต่อผู้ใช้) เจ้าของต้องตั้งนโยบายและยอมเสียรายได้บางส่วนเพื่อความน่าเชื่อถือ)

- **โมเดล/ทำไมคนกลับมา:** รวบรวมลิสต์สินค้า 'กว่า 50,000,000 รายการ' จากร้านค้าหลายเจ้า (Shopee/Lazada/ร้านแบรนด์) แล้วให้ผู้ใช้ค้น-เทียบราคาข้ามร้านในหน้าเดียว ปัจจุบันขยายเป็น 3 ขา: affiliate/Shopfluencer, Priceza Money (ประกัน/สินเชื่อ/บัตรเครดิต), และ E-commerce Insights (ขายข้อมูล/รายงานตลาด). เป็นเจ้าตลาดไทยตัวจริง อ้าง ~10 ล้านผู้ใช้/เดือน และเป็นแหล่งอ้างอิงตัวเลขตลาด e-commerce (มูลค่า 1.1-1.2 ล้านล้านบาท) ที่สื่อไทยหยิบไปใช้ต่อ  
  _เป็น default ในหัวคนไทยเวลาจะเทียบราคาก่อนซื้อ (brand recall 10+ ปี) + Hot deals มี countdown timer/ป้าย %ลด ทำให้กลับมาเช็คดีลซ้ำ + คอนเทนต์ Priceza Money (YouTube/TikTok) ดึงคนช่วงจะซื้อประกันรถ_
- **กลไกลายเซ็น:**
  - **เทียบราคาข้ามร้านบนสเกล catalog ยักษ์ (50M+ listings)** — หน้าเดียวรวมราคาจากหลายร้านต่อ 1 สินค้า พร้อมป้าย 'ราคาต่ำสุด' และแสดงราคาเดิม vs ราคาลด · *ทำไมได้ผล:* ความครบของ catalog = network effect: ร้านมาลงเพราะมีคนดู คนมาดูเพราะมีร้านครบ ทำให้คู่แข่งใหม่ไล่ตามยาก
  - **Adjusted CPC (aCPC) affiliate model** — เก็บเงินร้านค้าแบบ 'รายได้ต่อคลิก' ที่ปรับตามอัตราการซื้อจริง/มูลค่าสินค้า ไม่ใช่ CPC ตายตัว · *ทำไมได้ผล:* ปรับ price ตามคุณภาพทราฟฟิก ทำให้ร้านยอมจ่ายสูงขึ้นในหมวดที่ปิดการขายดี และ Priceza ได้ margin ดีกว่า flat CPC
  - **Priceza Money = ต่อยอด lead-gen การเงินจากทราฟฟิกเดิม** — วางตัวเป็น 'กลาง' เทียบประกันรถ/EV/เดินทาง/สินเชื่อ/บัตร ไม่ขายเป็นโบรกเกอร์ตรง แต่ได้ค่าคอมเมื่อผู้ใช้กดซื้อ · *ทำไมได้ผล:* หมวดการเงินมี CPL/commission สูงกว่าสินค้าอุปโภคหลายเท่า เอาทราฟฟิก intent สูงเดิมมาแปลงเป็นรายได้ต่อหัวที่แพงกว่าเดิมมาก
  - **E-commerce Insights เป็น trust + PR engine** — ออกรายงานมูลค่าตลาด/เทรนด์ที่สื่อ (Bangkokbiznews, Marketeer, longtunman) หยิบไปอ้าง · *ทำไมได้ผล:* สร้าง authority ฟรีทั้งวงการ ตอกย้ำภาพ 'เจ้าตลาด' และดัน backlink/SEO โดยไม่ต้องซื้อสื่อ
- **Trust device:** แสดงเลขทะเบียนพาณิชย์อิเล็กทรอนิกส์ 0105554078566 + ที่อยู่จริง (ERGO Tower กรุงเทพ); อ้าง track record '10+ ปี' และ presence 6 ประเทศ SEA; star ratings x.x/5 พร้อมจำนวนผู้รีวิวบนการ์ดสินค้า; ถูกอ้างอิงเป็นแหล่งตัวเลขตลาดโดยสื่อกระแสหลัก = social proof เชิงสถาบัน
- **รายได้:** aCPC/commission จาก affiliate + ค่าคอม/ค่าโฆษณาจากบริษัทประกัน-สินเชื่อผ่าน Priceza Money + ขายรายงาน/insights + โฆษณาบนเว็บ
- **จุดอ่อน/ความเสี่ยง:** ชั้น 'รีวิวที่คนเชื่อ' อ่อน: มี star rating รวม ๆ แต่ไม่มี hands-on testing/บรรณาธิการ ทำให้เป็น catalog ราคา ไม่ใช่ที่ตัดสินใจเชิงคุณภาพ; โดน Shopee/Lazada/TikTok ทำ in-app search + affiliate/voucher ในแอปตัวเอง กินฟังก์ชันเทียบราคาไปเรื่อย ๆ (disintermediation); ราคาที่โชว์อาจ lag/ไม่ตรงกับหน้าร้านจริงตอนกดเข้าไป ลดความน่าเชื่อถือ; พึ่ง affiliate feed จากแพลตฟอร์มที่เป็นทั้งพันธมิตรและคู่แข่ง — เสี่ยงถูกตัด/ลดค่าคอมเมื่อไรก็ได้
- **ปรับมาใช้กับ Friendsay:**
  - เพิ่ม 'ชั้นรีวิวที่ Priceza ไม่มี' เป็นจุดขายของเว็บรีวิว affiliate ไทย: ดึง feed ราคา/ลิงก์ซื้อแบบ Priceza แต่ครอบด้วย verdict บรรณาธิการ + ตารางสเปกเทียบ → หน้า 'X รุ่นน่าซื้อ 2026' ที่แต่ละรุ่นมีปุ่มเทียบราคา Shopee/Lazada สด ๆ + กล่อง 'เราแนะนำเพราะ...' 3 บรรทัด + ตารางสเปก (effort med, ติดที่: ต้องมี affiliate feed/API ราคาหลายร้าน (Involve Asia/Accesstrade) + คนเขียน verdict)
  - ลอกโมเดล lead-gen การเงินต่อหัวแพง (Priceza Money) มาต่อยอดหมวดที่เกี่ยวข้องกับสินค้าที่รีวิว → รีวิวมือถือ/รถ/เครื่องใช้ไฟฟ้าราคาสูง แล้วแทรกบล็อก 'ผ่อน 0%/ประกันเครื่อง/ประกันรถ' เป็น affiliate การเงิน (effort med, ติดที่: ต้องสมัคร partner การเงิน (บัตร/ประกัน/ผ่อน) ซึ่งอนุมัติช้าและต้องมีทราฟฟิกระดับหนึ่ง)
  - สร้าง insight/report ประจำปีของ niche ตัวเองเลียนแบบ E-commerce Insights เพื่อดึง PR + backlink → รายงาน 'สรุปราคา-เทรนด์หมวด[สินค้าของเรา] ปี 2026' ปล่อยฟรี ให้สื่อ/เพจอ้างอิง (effort low, ติดที่: ต้องสะสม data ราคาย้อนหลังของตัวเองพอสมควรก่อนถึงจะน่าเชื่อ)

### BigGo Thailand (biggo.co.th)
- **ประเภท:** Price Smart Search Engine สายไต้หวัน เน้น price-history + anti-fake-discount + shopping assistant
- **โมเดล/ทำไมคนกลับมา:** เสิร์ชเอนจินเทียบราคาข้ามแพลตฟอร์ม (Shopee ฯลฯ) ที่ต่างจากคนอื่นตรง 'ประวัติราคาย้อนหลัง 2 ปี' + แจ้งเตือนราคาตกแบบเรียลไทม์ ผลักดันผ่าน browser extension + แอปมือถือเป็นหลัก (mobile-first) พร้อมฟีเจอร์ AI: ค้นด้วยรูป BIRSE, ตรวจคอมเมนต์โซเชียล, generator คำโปรย, คูปอง/แคชแบ็ก  
  _ตั้ง price alert แล้วระบบดึงกลับมาเองเวลาราคาตก (retention เชิงระบบ ไม่ต้องจำเข้าเว็บ) + ก่อนกดซื้อช่วงเทศกาลคนเปิดกราฟ 2 ปีเช็คว่า 'ลดจริงหรือขึ้นก่อนลด' + extension ทำงานคาหน้า Shopee เลย_
- **กลไกลายเซ็น:**
  - **กราฟประวัติราคาย้อนหลัง 2 ปี** — หน้าสินค้าโชว์เส้นราคาย้อนหลังถึง 2 ปี ให้เห็นจุดต่ำสุด/สูงสุด · *ทำไมได้ผล:* ให้ 'หลักฐานเชิงตัวเลข' ว่าควรซื้อตอนนี้ไหม เปลี่ยนการตัดสินใจจากอารมณ์เป็นข้อมูล = โมเดลเดียวกับ CamelCamelCamel ที่สร้างความผูกพันสูง
  - **จับโปรลวง 'ขึ้นก่อนลด' (fake-discount detection)** — ใช้ประวัติราคาเผยว่าราคาถูกดันขึ้นก่อนเทศกาลแล้วค่อยลด ป้ายลดเลยไม่จริง · *ทำไมได้ผล:* โจมตี pain point ที่คนช้อปช่วง 11.11/12.12 เกลียดที่สุด สร้างภาพ 'อยู่ข้างผู้ซื้อ' — trust ที่ Priceza/iPrice ให้ไม่ได้
  - **Price alert เรียลไทม์ผ่าน app/extension** — subscribe สินค้าแล้วเด้งแจ้งเตือนทันทีเมื่อราคาลด · *ทำไมได้ผล:* สร้าง retention loop อัตโนมัติ + เก็บ intent data ว่าใครจ้องสินค้าอะไร เอาไปยิงดีล/แคชแบ็กได้ตรงจุด
  - **AI shopping assistant: BIRSE image search + ตรวจคอมเมนต์** — ค้นสินค้าด้วยรูป, ดึงคอมเมนต์โซเชียลมาสรุป, สร้างคำอธิบายสินค้า · *ทำไมได้ผล:* เพิ่ม entry point นอกเหนือจากพิมพ์คีย์เวิร์ด และดึงคนสาย visual/สาย 'อยากรู้ว่าคนพูดถึงยังไง' เข้ามาในระบบ
- **Trust device:** กราฟราคา 2 ปี = หลักฐานตัวเลขที่ตรวจสอบได้ (ไม่ใช่คำโฆษณา); จุดยืน 'ช่วยจับโปรลวง' วางตัวเป็นฝ่ายผู้บริโภค; cross-platform coverage (Shopee, Momo, Rakuten, Booking, Klook ฯลฯ) แสดงความเป็นกลาง; browser extension เรตติ้ง/รีวิวบน Chrome/Firefox store เป็น social proof

- **จุดอ่อน/ความเสี่ยง:** แบรนด์ยังไม่ติดหัวคนไทยเท่า Priceza — ต้องพึ่ง extension/แอปในการ acquire ซึ่ง friction สูงกว่าเปิดเว็บ; ไม่มีชั้นรีวิว/testing เชิงคุณภาพเช่นกัน — ตอบได้แค่ 'ราคาดีไหม' ไม่ตอบ 'รุ่นไหนดีกว่า'; ความแม่นของ price history ขึ้นกับความถี่ crawl; ถ้าเก็บไม่ครบ กราฟจะทำให้เข้าใจผิดได้; แพลตฟอร์มต้นทาง (Shopee) อาจบล็อก crawler หรือฟ้องเรื่องดึงข้อมูล

### iPrice Thailand (ipricethailand.com)

- **โมเดล/ทำไมคนกลับมา:** ระดับกลุ่ม iPrice มี 3 ขา (เทียบราคา electronics/beauty, product discovery แฟชั่น/บ้าน, คูปองทุกหมวด) แต่ ณ 2026 หน้า ipricethailand.com ที่เปิดจริง = หน้ารวมคูปอง/โค้ดส่วนลด (Klook, Shopee, Agoda, Samsung, Lazada, GrabFood, Traveloka) ไม่ได้เด่นเรื่องเทียบราคาสินค้าต่อชิ้นแล้ว. ถูก Bukalapak (อินโดฯ) ซื้อหุ้นใหญ่ เม.ย. 2023 และบริษัทแม่เพิ่งถอยจากสินค้า physical (หยุดรับออเดอร์ physical 9 ก.พ. 2025) + เคยเลย์ออฟ ~20%

- **กลไกลายเซ็น:**

  - **โครง SEO landing ต่อ merchant + แคมเปญตามฤดู** — หน้า mid-month sale / travel deal / ร้านอาหาร พร้อมตัวเลขส่วนลดเจาะจง (%/บาท) · *ทำไมได้ผล:* ดัก long-tail keyword ตามชื่อร้าน/เทศกาล ได้ traffic organic ต้นทุนต่ำแบบต่อเนื่อง

- **รายได้:** ค่าคอมมิชชั่น LINE MAN delivery, ดีล/ส่วนลด (Wongnai Deal), บริการร้านค้าแบบ subscription/POS/CRM (WMA), โฆษณา/ตำแหน่งเด่นของร้าน, และ data/insight

## ไทย · content-commerce / รีวิวสายเทค+ของใช้


### Mercular.com
- **ประเภท:** Content-commerce hybrid — owns the store AND the editorial (reviews/listicles feed its own checkout, not a third party)
- **โมเดล/ทำไมคนกลับมา:** Product margin as an authorized reseller of audio/gadgets, not affiliate commission. Editorial ('best-of', 'review-article', Editors' Pick) exists to funnel intent-stage readers into its own cart. This is the key structural difference from every pure-affiliate Thai site: Mercular captures full retail margin, so it can invest far more per article and still profit.  
  _Trust bundle: buyers come back because it is a real store that solves the No.1 Thai fear (fake/grey-market goods) — official reseller, sealed authentic units, Thai-dealer warranty, free 7-day claim, 0% installment up to 10 months, free shipping over 199 baht. The editorial makes it feel like advice, the store makes it safe to act._
- **กลไกลายเซ็น:**
  - **Ranked 'best-of / X ยี่ห้อไหนดี' listicles with numbers in the title** — Templated pages like 'แนะนำ 12 อันดับ หูฟังไร้สาย', 'แนะนำ 16 หูฟังบลูทูธ', '/best-of/best-of-true-wireless' — segmented by use-case, price band, and feature (bass-heavy, flat-cable, with-mic). · *ทำไมได้ผล:* Matches exactly how Thais search intent-stage queries ('ยี่ห้อไหนดี'), and every recommended item is an in-stock SKU one click from cart — zero leakage to another site.
  - **Consult-to-convert layer** — Explicit 'ยินดีให้คำปรึกษา/เปรียบเทียบสินค้า' plus a LINE/chat sales team, framing the site as an advisor rather than a shelf. · *ทำไมได้ผล:* High-consideration audio/gadget buyers want reassurance before spending; a human close reduces cart abandonment that pure blogs can't recover.
  - **Owned video review series feeding the same catalog** — 'Mercular Editors' Pick' YouTube series (e.g. EP.2 on top true-wireless) mirrors the on-site listicles. · *ทำไมได้ผล:* Same catalog, two funnels (search + video). Video builds face/brand trust that text reviews lack, and description links go to its own store.
  - **It also runs its OWN affiliate program (Involve Asia)** — Recruits outside creators to push Mercular links for commission — so Mercular is simultaneously merchant, publisher, and affiliate-network node. · *ทำไมได้ผล:* Turns competitors' audiences into a distributed sales force while it keeps the margin; a pure-affiliate blog can never do this.
- **Trust device:** 'เราคือตัวแทนจำหน่าย' / authorized-reseller page + brand logos; Authentic-sealed-goods + Thai dealer warranty guarantee; Free claim within 7 days of receipt; 0% installment up to 10 months (lowers price objection); Free nationwide shipping over 199 baht; 7+ year operating history stated as a credibility anchor
- **รายได้:** Direct retail margin on owned inventory (primary) + its own affiliate program recruiting external creators + on-site promotions/clearance events.
- **จุดอ่อน/ความเสี่ยง:** Editorial is inherently conflicted — every 'best' is something Mercular stocks, so rankings can't credibly include items it doesn't sell.; No visible independent testing methodology or 'we measured X' proof — recommendations rest on brand authority, not data.; Category-locked to audio/gadgets; limited authority outside that vertical.; Requires real capital/logistics (inventory, warranty, returns) — not a model a small review blog can copy directly.
- **ปรับมาใช้กับ Friendsay:**
  - Adopt the number-in-title, use-case-segmented 'ยี่ห้อไหนดี' listicle grid, but make each pick a transparent affiliate link with a visible disclosure Mercular can't offer → Templated Astro collection: '12 อันดับ [หมวด] ยี่ห้อไหนดี 2026' segmented by price band + use-case, each card = spec chips + Shopee/Lazada buttons + an explicit 'ลิงก์นี้เป็น affiliate' badge and a 'ทำไมติดอันดับ' one-liner. (effort low, ติดที่: none — template + existing 6-category catalog)
  - Add a 'consult-to-convert' human layer (Mercular's quiet moat) via LINE OA quick-help on high-consideration pages → Sticky 'ถามก่อนซื้อ' LINE button on comparison pages; owner or a rota answers 'รุ่นไหนเหมาะกับผม' and drops the affiliate link. (effort med, ติดที่: owner time/staffing to answer — decide if worth it per category)
  - Beat Mercular on the one thing it structurally cannot do: neutrality. Publish an explicit 'เราไม่ได้ขายของเอง จึงเลือกได้ทุกยี่ห้อ' promise → A standing trust page + a badge on every roundup stating Friendsay stocks nothing, discloses every affiliate link, and can therefore recommend brands a store-owner would never rank. (effort low, ติดที่: none — copywriting + policy consistency)

### Beartai (BT beartai) / Droidsans (media-review archetype)
- **ประเภท:** Large tech-media publisher — editorial roundups + individual reviews with huge social/video reach, monetized by ads/sponsorship rather than a tight buy funnel
- **โมเดล/ทำไมคนกลับมา:** Audience-at-scale sold to advertisers: display, sponsored reviews/PR, brand events, and loose Shopee/YouTube-affiliate tagging. Reach is the product; commerce conversion is a weak afterthought. (Droidsans shows the scale: ~688k FB, TikTok 1.2M followers / 41.9M likes.)  
  _News cadence + trusted long-running brand + personalities. Readers come for daily tech news and launch coverage, and reviews ride that traffic. It's habit/brand loyalty, not a shopping tool._
- **กลไกลายเซ็น:**
  - **Editorial 'flagship roundup' with real starting prices** — e.g. '9 เรือธงที่รีวิวปี 2025' listing each phone with a THB starting price (HUAWEI Mate XT 119,990฿, Galaxy S25 Ultra 46,900฿) + a 2–4 sentence feature story each. Notably: no buy buttons, no scores, no comparison table — pure browse-and-read. · *ทำไมได้ผล:* Timely, SEO-friendly, and shareable; captures 'what launched / what's hot' demand cheaply.
  - **Individual reviews with จุดเด่น/จุดด้อย and concrete specs+price** — Droidsans-style single-device reviews (e.g. Galaxy A06 5G, Dimensity 6300, เริ่ม 4,499฿) that name real weaknesses (thick bezels, LCD). · *ทำไมได้ผล:* Naming flaws builds credibility that PR-only sites lack; specific price/chip detail satisfies spec-hunters.
  - **Cross-platform video/short-form engine** — YouTube long reviews + TikTok shorts ('มือถืองบ 3000', 'มือถือตัวโหดราคาดิ่ง') drive discovery independent of Google. · *ทำไมได้ผล:* Short-form is where Thai purchase discovery now happens; a face on camera converts trust faster than text.
- **Trust device:** Long-established brand name / newsroom authority; Willingness to state cons in reviews; Named on-camera reviewers (personalities as credibility); Large visible social proof (follower/like counts)
- **รายได้:** Display ads + sponsored/PR reviews + brand events + YouTube/TikTok monetization + loose Shopee/YouTube affiliate tags. Conversion tracking is weak; affiliate is not the core.
- **จุดอ่อน/ความเสี่ยง:** Roundups have no buy buttons/no scores — massive intent leaks out with no capture or commission.; Sponsored-review model creates an undisclosed-bias perception; readers can't tell paid from independent.; No structured comparison or testing data — 'facts in comparison' style rigor is absent.; Depends on ad rates + platform algorithms; commerce upside is under-monetized.
- **ปรับมาใช้กับ Friendsay:**
  - Do what Beartai's roundups omit: bolt a buy-button + live-price row onto every roundup item so intent is captured, not leaked → On each '[หมวด] ยี่ห้อไหนดี' item card: current price + Shopee/Lazada affiliate buttons + a 'ราคาอัปเดตล่าสุด [date]' stamp. (effort low, ติดที่: none for manual price; automated pricing feed = med and blocked by data source)
  - Win the trust gap Thai media create with paid reviews: a per-article disclosure/independence banner → Standard header on every review: 'บทความนี้ไม่ได้รับสปอนเซอร์ / ได้รับสินค้าตัวอย่าง — เปิดเผยตรงนี้', plus a persistent 'ไม่รับเงินจัดอันดับ' policy. (effort low, ติดที่: none — policy + template)
  - Copy their short-form discovery engine but link each clip to a structured on-site comparison, not just a feed → TikTok/Shorts hook ('X vs Y รุ่นไหนคุ้ม 2026') → link in bio to the Friendsay comparison page with the affiliate buttons. (effort med, ติดที่: owner capacity to produce video consistently)

### Priceza.com (price-comparison decision engine)
- **ประเภท:** Aggregator / price-comparison engine — the Thai 'help me decide by price' utility, monetized by CPC/affiliate redirects to stores
- **โมเดล/ทำไมคนกลับมา:** Aggregates 50,000,000+ listings and sends buyers out to Lazada/Shopee/merchants via 'เช็คราคา' redirect buttons, earning CPC/affiliate on the handoff. It owns no inventory and writes little editorial — it monetizes the comparison moment itself.  
  _It's a tool, not a read: shoppers return at the final decision step to check 'ที่ไหนถูกสุด' before buying. Utility + breadth of coverage = habitual bottom-of-funnel visits._
- **กลไกลายเซ็น:**
  - **Massive multi-store price aggregation** — 'ฐานข้อมูลมากกว่า 50,000,000 รายการ' compared across many merchants, each product showing seller name + 'เช็คราคา' redirect. · *ทำไมได้ผล:* Breadth is the moat — one query surfaces every store, which no single review blog can match, so it owns the final price-check.
  - **Discount-percentage badges + original-vs-now pricing** — Visible 'ลด 26%', 'ลด 31%' badges with struck original price beside current price. · *ทำไมได้ผล:* Quantified saving triggers urgency and makes the 'deal' legible in one glance.
  - **Star rating + review-count on product cards** — e.g. '4.3/5 (2021)' — aggregated rating with sample size shown. · *ทำไมได้ผล:* Sample size next to the score is a cheap, high-trust signal (crowd proof) that turns a price list into a decision aid.
  - **Price-history / lowest-price framing (the CamelCamelCamel-style hook)** — Priceza is known for price-tracking/history on product detail pages and a 'ราคาต่ำสุด' framing, letting buyers judge if now is a good time. · *ทำไมได้ผล:* Answers the real anxiety ('is this actually cheap right now?') that a static review never addresses — this is the single most transferable mechanic to Friendsay.
- **Trust device:** Sheer coverage count (50M+ items) as authority signal; Transparent seller name on every price row; Aggregated star rating WITH review count; Quantified discount %/lowest-price references
- **รายได้:** CPC + affiliate commission on outbound clicks to Lazada/Shopee/merchants; merchant listing/ad placements.
- **จุดอ่อน/ความเสี่ยง:** Commodity utility — thin/no genuine editorial or hands-on testing, so it can't win 'which is actually better', only 'which is cheaper'.; Aggregated ratings are borrowed from marketplaces (gameable, not independently verified).; Price data can go stale; a wrong 'lowest price' erodes trust fast.; Little brand affinity — used transactionally, easily displaced by whoever has fresher data or better UX.
- **ปรับมาใช้กับ Friendsay:**
  - Add a price-history + 'ควรซื้อตอนนี้ไหม' verdict block — the CCC/Priceza mechanic that Thai review blogs almost never have → On each product/comparison page: a small price-over-time chart (or at minimum 'ต่ำสุดที่เคยเห็น X บาท / ราคาปัจจุบัน Y') + a 'ราคานี้ถือว่า ถูก/ปกติ/แพง' badge, refreshed on a schedule. (effort high, ติดที่: needs a price-history data pipeline (scrape/API + storage) — blocked on data source and owner infra decision)
  - Show rating WITH sample size on every recommended item, but label the source to out-transparent Priceza's borrowed ratings → Card chip: '⭐4.3 · จาก 2,021 รีวิว Shopee' — explicitly sourced, so it reads as honest aggregation rather than an unlabelled score. (effort low, ติดที่: rating data availability per SKU (manual entry low, automated med))
  - Stamp freshness on every price to beat Priceza's staleness risk and Thai blogs' rot → 'ราคาอัปเดต 16 ก.ค. 2026' badge on each buy button; auto-flag/hide prices older than N days so users trust the number. (effort med, ติดที่: requires a price-refresh routine; must survive View-Transitions client-nav per repo constraint)
