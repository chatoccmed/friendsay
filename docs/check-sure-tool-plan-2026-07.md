# แผนเครื่องมือ "Friendsay เช็กชัวร์" — เอกซเรย์สินค้าก่อนกดซื้อ

จัดทำ: 3 กรกฎาคม 2026 · สถานะ: รอผลตรวจสอบจากทีมรีวิว → รอเจ้าของอนุมัติเริ่มเฟส 1

## แนวคิด

นักช้อปยุคนี้ตรวจ 7 อย่างก่อนซื้อ (ยอดขาย / จำนวนรีวิว / ถูกสุดไหม / โค้ดลดเหลือเท่าไหร่ / รีวิวดีบอกอะไร / รีวิวแย่บอกอะไร / ภาพครบไหม) — งาน 15 นาทีต่อชิ้นที่ทุกคนจำใจทำเอง "เช็กชัวร์" ย่อให้เหลือ 15 วินาที ด้วยใบเอกซเรย์ 1 ใบต่อสินค้า สร้างจาก evidence ledger ที่เราเก็บจริงอยู่แล้ว

จุดที่ลอกยาก (moat): ข้อมูลทุกช่องมาจากการเปิดหน้าสินค้าจริง มีวันที่กำกับ ตรวจย้อนได้ — AI farm ไม่มีข้อมูล / เว็บใหญ่ไม่เก็บมือ / Shopee ไม่มีแรงจูงใจโชว์ "เคยถูกกว่านี้" กับ "รีวิวแย่บอกอะไร"

## โมดูลในใบเอกซเรย์ (ทุกข้อมี data source แล้ว)

1. ยอดขายที่เห็นจริง + เทียบค่ากลางหมวด
2. จำนวนรีวิว + สัดส่วนดาว (โดนัทกราฟ 5★→1★)
3. Price history: ราคาวันนี้ vs ที่เราเคยเห็น (ต่ำสุดในรอบที่ติดตาม?)
4. โค้ดร้านที่เห็น ณ วันที่ + ราคาสุทธิหลังโค้ด + ปฏิทินแคมเปญ
5. รีวิวดีพูดซ้ำเรื่องอะไร (พร้อมจำนวน)
6. รีวิวแย่พูดเรื่องอะไร (สัดส่วน 1★ + ธีม) ← จุดขายความซื่อสัตย์
7. เช็กลิสต์ภาพ: ภาพจริง/วิดีโอ/รีวิวมีสื่อกี่รายการ
8. แถม: ความน่าเชื่อถือร้าน (Mall/ตอบแชท%) + สถานะส่ง (จับพรีออเดอร์) + ทางเลือกที่ดีกว่าในงบเดียวกัน (affiliate)

กลไกเติบโต: ช่องวางลิงก์/ค้นหา → ถ้าไม่อยู่ในฐาน → ปุ่ม "ขอให้ Friendsay ตรวจ" = ผู้ใช้บอกเราเองว่าควรเพิ่มสินค้าไหน (demand-driven content queue)

## รายได้

1. Affiliate ณ จุด intent สูงสุด (ปุ่มซื้อ + ทางเลือกที่ดีกว่า)
2. LINE price-drop alert (เฟส 3) — trigger ที่ conversion สูงสุด + เก็บ Indirect 0.6%
3. SEO/AEO: ใบเอกซเรย์ = หน้า landing ของคำค้น "รุ่นX ดีไหม/ลดราคาไหม"

## งานที่ต้องทำ

### เฟส 1 — ใบเอกซเรย์บนเว็บ (เริ่มได้ทันที)

- [ ] 1.1 โครงข้อมูล price history: ตาราง log (productKey, date, price, source) + เติมย้อนหลังจากข้อมูลที่เก็บแล้ว (เช่น HAFELE 2,690→2,434 ในวันเดียว)
- [ ] 1.2 โครงข้อมูลใบเอกซเรย์: ขยาย proof schema ให้ครบ 8 โมดูล (star breakdown, โค้ดที่เห็น, media checklist, shop trust)
- [ ] 1.3 คอมโพเนนต์ "ใบเอกซเรย์" (ผ่านมาตรฐาน design system: กราฟโดนัท + แถบเทียบ + motion)
- [ ] 1.4 หน้า /th/tools/check-sure/ + ช่องค้นหา/วางลิงก์จับคู่กับฐาน verified
- [ ] 1.5 ปุ่ม "ขอให้ Friendsay ตรวจ" → เก็บคำขอ (เริ่มจาก mailto/LINE ลิงก์ ก่อนมีฟอร์มจริง)
- [ ] 1.6 ฝังใบเอกซเรย์ย่อในหน้ารีวิว/roundup ที่มีอยู่ (reuse ทันที 17+ สินค้า)

## ความเสี่ยง/ข้อจำกัดที่ประกาศไว้ก่อน

- เฟส 1 ครอบคลุมเฉพาะสินค้า verified (~17 ตัว โต ~10/สัปดาห์) — จุดขายคือ "ตรวจจริงทีละตัว" ไม่ใช่ครอบคลุมทุกอย่าง
- ราคา/โค้ดเปลี่ยนเร็วกว่ารอบเก็บ → ทุกตัวเลขต้องมีวันที่กำกับ + ปุ่มเช็กราคาปัจจุบันเสมอ
