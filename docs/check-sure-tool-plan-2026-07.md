# แผนเครื่องมือ "Friendsay เช็กชัวร์" — เอกซเรย์สินค้าก่อนกดซื้อ

จัดทำ: 3 กรกฎาคม 2026 · สถานะ: รอผลตรวจสอบจากทีมรีวิว → รอเจ้าของอนุมัติเริ่มเฟส 1

## แนวคิด

นักช้อปยุคนี้ตรวจ 7 อย่างก่อนซื้อ (ยอดขาย / จำนวนรีวิว / ถูกสุดไหม / โค้ดลดเหลือเท่าไหร่ / รีวิวดีบอกอะไร / รีวิวแย่บอกอะไร / ภาพครบไหม) — งาน 15 นาทีต่อชิ้นที่ทุกคนจำใจทำเอง "เช็กชัวร์" ย่อให้เหลือ 15 วินาที ด้วยใบเอกซเรย์ 1 ใบต่อสินค้า สร้างจาก evidence ledger ที่เราเก็บจริงอยู่แล้ว

จุดที่ลอกยาก (moat): ข้อมูลทุกช่องมาจากการเปิดหน้าสินค้าจริง มีวันที่กำกับ ตรวจย้อนได้ — AI farm ไม่มีข้อมูล / เว็บใหญ่ไม่เก็บมือ / Shopee ไม่มีแรงจูงใจโชว์ "เคยถูกกว่านี้" กับ "รีวิวแย่บอกอะไร"

## โมดูลในใบเอกซเรย์ (ทุกข้อมี data source แล้ว)

1. ยอดขายที่เห็นจริง + เทียบค่ากลางหมวด
2. จำนวนรีวิว + สัดส่วนดาว (โดนัทกราฟ 5★→1★)
3. Price history: ราคาวันนี้ vs ที่เราเคยเห็น (ต่ำสุดในรอบที่ติดตาม?)
4. โค้ดร้านที่เห็น ณ วันที่ + ราคาสุทธิหลังโค้ด + ปฏิทินแคมเปญ
5. รีวิวดีพูดซ้ำเรื่องอะไร (พร้อมจำนวน)
6. รีวิวแย่พูดเรื่องอะไร (สัดส่วน 1★ + ธีม) ← จุดขายความซื่อสัตย์
7. เช็กลิสต์ภาพ: ภาพจริง/วิดีโอ/รีวิวมีสื่อกี่รายการ
8. แถม: ความน่าเชื่อถือร้าน (Mall/ตอบแชท%) + สถานะส่ง (จับพรีออเดอร์) + ทางเลือกที่ดีกว่าในงบเดียวกัน (affiliate)

กลไกเติบโต: ช่องวางลิงก์/ค้นหา → ถ้าไม่อยู่ในฐาน → ปุ่ม "ขอให้ Friendsay ตรวจ" = ผู้ใช้บอกเราเองว่าควรเพิ่มสินค้าไหน (demand-driven content queue)

## รายได้

1. Affiliate ณ จุด intent สูงสุด (ปุ่มซื้อ + ทางเลือกที่ดีกว่า)
2. LINE price-drop alert (เฟส 3) — trigger ที่ conversion สูงสุด + เก็บ Indirect 0.6%
3. SEO/AEO: ใบเอกซเรย์ = หน้า landing ของคำค้น "รุ่นX ดีไหม/ลดราคาไหม"

## ผลตรวจสอบจากทีมรีวิว 3 มุมมองอิสระ (2026-07-03)

| ผู้ตรวจ | คำตัดสิน | ประเด็นหลัก |
|---|---|---|
| นักกลยุทธ์ผลิตภัณฑ์ | GO-แบบปรับ | ใบเอกซเรย์ = component ของหน้า content ไม่ใช่หน้า tool; ทำข้อฝังในหน้าเดิมเป็นงานแรก; ดึง LINE ขึ้นมา; บังคับ analytics ก่อนปล่อย |
| นักการตลาด (โจทย์: ฆ่าไอเดีย) | NO-GO standalone / GO เฉพาะแบบฝัง | คอขวดจริงคือ distribution ไม่ใช่ feature — เว็บยังไม่มีถนนเข้า (โดเมนยังไม่ย้าย, ไม่มี analytics, ไม่มีช่องทางกระจาย); ช่องวางลิงก์ล้มเหลว 99% ตั้งแต่ครั้งแรกเพราะฐาน 17 ตัว; hook ที่แรงสุด = "สิ่งที่ Shopee ไม่มีวันบอก: เคยถูกกว่านี้ + รีวิว 1 ดาวบ่นอะไร" ในรูป**ภาพแชร์ได้** |
| วิศวกรอาวุโส | เฟส 1 GO-แบบปรับ / เฟส 2 ลดสเปกหนัก / เฟส 3 สลับลำดับ | จับคู่ลิงก์ทำได้จริงด้วย shopid.itemid ใน URL (inline index ไม่ต้อง backend); bookmarklet overlay ขัดบทเรียน pipeline ตัวเอง เหลือได้แค่ตัวส่งต่อ 1 บรรทัด; price history = CSV append-only แยก list/promo; **ใบเอกซเรย์ต้อง fail-safe เอง: ข้อมูลเกิน 14 วันลดรูปอัตโนมัติ** |

### ฉันทามติ (ทั้ง 3 คนพูดตรงกันโดยไม่เห็นกัน)

1. **ฝังในหน้าที่มีอยู่ก่อน (ข้อ 1.6 เดิม) คืองานแรก ไม่ใช่งานท้าย** — พิสูจน์คุณค่ากับ traffic ที่มีจริง ไม่ต้องรอใคร
2. **ตัดหน้า /tools/ + ช่องวางลิงก์ออกจากเฟสแรก** — กลับมาทำเมื่อฐาน verified ~150+ และมี traffic
3. **ฆ่า bookmarklet overlay** — ตลาดไทยอยู่ในแอปมือถือ; เหลือไว้แค่ตัวส่งต่อ 1 บรรทัดในอนาคต
4. **Price-drop alert ต้องรอ API** (เมนู "เปิด API" ใน dashboard) — สัญญา alert บนรอบเก็บมือรายสัปดาห์ = anti-feature
5. **ก่อนปล่อยต้องมี analytics + sub_id=checksure** — เมตริกตัดสิน 30 วัน: คลิก affiliate จากใบเอกซเรย์ ≥25% ของหน้าที่ฝัง = ลงทุนต่อ / <10% = หยุด
6. คอขวดอันดับหนึ่งของทั้งเว็บตอนนี้คือ **ย้ายโดเมน + analytics + ช่องทางกระจาย 1 ช่อง** — งานเครื่องมือมาหลังจากนั้น

### ข้อเท็จจริงสำคัญจากเจ้าของ (2026-07-03)

**Shopee Affiliate Open API ต้องมีแอป/เครื่องมือที่ใช้งานได้จริงก่อน ถึงจะยื่นขอสมัครได้** — เปลี่ยนกลยุทธ์เป็น bootstrap 2 จังหวะ:

1. **v1 "ค้นจากของที่ตรวจแล้ว"** — หน้าค้นหาโล่ง ๆ ตามวิสัยทัศน์เจ้าของ (พิมพ์ "แอร์ราคาต่ำกว่า 10000" → top 5 เรียงยอดขาย พร้อมรูป+สรุป+ลิงก์ affiliate) แต่ขุมข้อมูล = ฐาน verified ของเรา (pre-baked ตอน build, ค้นฝั่ง client, ไม่ต้องมี backend) — วางตำแหน่งตรง ๆ ว่า "ค้นเฉพาะของที่ Friendsay ตรวจหลักฐานแล้ว X รุ่น" เพื่อไม่ให้คนคาดหวังว่าค้นได้ทั้ง Shopee; ฐานโตทุกสัปดาห์ตามรอบเก็บ + คำขอผู้ใช้
2. **ยื่นขอ Open API โดยใช้ v1 + traffic เป็นผลงาน** → อนุมัติแล้วจึงสลับชั้นข้อมูลเป็น API สด (ค้นได้ทั้ง Shopee + alert อัตโนมัติ) = เวอร์ชันเต็มตามฝัน

LINE alert ช่วง v1: สื่อสารตามความสามารถจริงเท่านั้น — "เช็กราคาให้ทุกสัปดาห์ + ก่อนวันดับเบิ้ล" (ห้ามสัญญา real-time จนกว่าจะมี API)

### รอเงื่อนไข

- หน้า /tools/check-sure/ + ช่องวางลิงก์ (กลไก shopid.itemid inline index ตามที่วิศวกรออกแบบ) → เมื่อฐาน ≥150 + traffic วันละ 300+
- Bookmarklet ส่งต่อ 1 บรรทัด + flow มือถือ (แชร์→คัดลอก→วาง) → พร้อมหน้า /tools/
- LINE price-drop alert → หลังสำรวจ Affiliate API (3.2) แล้วดึงราคาอัตโนมัติได้จริง
