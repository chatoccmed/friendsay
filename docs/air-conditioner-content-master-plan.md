# แผนแม่บทคอนเทนต์หมวดแอร์ — Friendsay Air Authority Plan

จัดทำ: 2 กรกฎาคม 2026
เป้าหมาย: ภายใน 3 เดือน Friendsay เป็นเว็บภาษาไทยที่ตอบคำถามเรื่อง "เลือกซื้อแอร์" ได้ครบ loop ที่สุด — ตั้งแต่ยังไม่รู้จะเริ่มตรงไหน จนถึงกดซื้อและนัดช่างเสร็จ

## นิยาม "ผู้เชี่ยวชาญด้านแอร์" ที่วัดได้

เว็บจะถือว่าเป็น authority หมวดแอร์เมื่อ:

1. คำถามหลักทุกขั้นของ buyer journey มีหน้าเนื้อหารองรับ (ดูตารางหัวข้อ)
2. ทุกอันดับ/คำแนะนำ ย้อนกลับไปหาหลักฐานใน evidence ledger ได้
3. มีเครื่องมือ interactive ที่เว็บอื่นไม่มี (BTU calculator, ค่าไฟ/ปี)
4. ราคา/ข้อมูลมีวันที่กำกับและอัปเดตทุกเดือน
5. โครง internal link เชื่อมครบ: guide → roundup → single review → tool → ปุ่มซื้อ

## สถาปัตยกรรมเนื้อหา (hub-and-spoke + tools)

```
Pillar: ศูนย์รวมเลือกซื้อแอร์ (/th/best/air-conditioners/ ยกระดับเป็น hub ใหญ่)
├── Roundups (AC-R) บทความรวมจัดอันดับตามธีม  ← ทางเข้า traffic หลัก
│     └── ลิงก์ลง Single reviews (มีแล้ว 25 คิว) + ปุ่มซื้อ
├── Guides (AC-G) คู่มือความรู้  ← สร้าง topical authority + ดัก AI search
│     └── ฝัง Tools + ลิงก์ไป roundup ที่เกี่ยว
├── Brand hubs (AC-B) รวมรุ่นรายแบรนด์  ← ดักคำค้น "ยี่ห้อ X รุ่นไหนดี"
├── Versus (AC-V) เทียบตัวต่อตัว  ← ดักคนใกล้ตัดสินใจสุด
├── Tools (AC-T) เครื่องมือ interactive  ← จุดต่าง + backlink magnet
└── Data (AC-D) หน้าราคา/ดีลอัปเดตประจำ  ← เหตุผลให้คนกลับมา
```

## กติกากลางก่อนผลิต (ห้ามข้าม)

- บทความรวมต้องมี ≥10 รุ่นที่ผ่าน `verified_sold` / `review_signal_pass` (ณ 2 ก.ค. 2026 มี 4 → ต้องเก็บ evidence เพิ่มก่อน)
- ทุกบทความมีกล่อง "เกณฑ์การจัดอันดับ + วันที่เก็บข้อมูล" และป้ายอัปเดตล่าสุด
- ห้ามคำว่า "ขายดีที่สุด" ถ้าไม่เห็น sold_count — ใช้ "น่าสนใจจากจำนวนรีวิวใน Shopee"
- คะแนนคิดจากสูตร Friendsay Air Score โดย re-weight ตามธีม และอันดับ render จาก data file ไม่เขียนมือ
- ทุกรุ่นที่ถูกจัดอันดับต้องมี "ไม่เหมาะกับใคร" ของตัวเอง
- guide เขียนได้เลยไม่ต้องรอ evidence (ไม่มีการจัดอันดับสินค้า) — เป็นงานที่เริ่มก่อนได้เสมอ

## Definition of Done รายประเภท

- **Roundup**: ≥10 รุ่นผ่านเกณฑ์, กล่องเกณฑ์+วันที่, quick-pick box บนสุด, ตารางเทียบ, การ์ดรายรุ่นมี "ไม่เหมาะกับ", ตัวช่วยเลือกตามสถานการณ์ท้ายบท, FAQ, ItemList schema, ลิงก์ลง single review, build ผ่าน
- **Guide**: ตอบคำถามหลักใน 90 วินาทีแรก, มีตาราง/ภาพ/เครื่องมือประกอบ, FAQ schema, ลิงก์ไป roundup ที่เกี่ยว, ไม่ขายของแข็ง (guide คือที่สร้างความเชื่อ ไม่ใช่ที่ยัดปุ่ม)
- **Tool**: ใช้ได้บนมือถือ, ผลลัพธ์แชร์/บันทึกได้, มี CTA ไป roundup ที่ตรงผลลัพธ์
- ทุกประเภท: ผ่าน QA checklist ใน product-review-operating-template.md + `npm.cmd run build` ผ่าน

## การวัดผล (ดูทุกสิ้นเดือน)

1. Search Console: จำนวนหน้า indexed, คำค้นที่ impression โต, CTR
2. Analytics: หน้าไหนคนอ่านจบ (scroll depth), ทางเข้า → เส้นทางคลิก
3. Affiliate: คลิกปุ่มซื้อ แยกตาม store/context (มี tracker อยู่แล้ว รอต่อปลายทาง)
4. เมื่อได้บัญชี affiliate: conversion + ค่าคอมจริง ต่อบทความ → เทน้ำหนักผลิตไปหาธีมที่ทำเงิน
