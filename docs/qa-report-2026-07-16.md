**False positives ที่ยืนยันแล้ว (ไม่ใช่ปัญหา):**
1. `/site.webmanifest` — regex ตัด ext ยาว >5 ตัวพลาด · ไฟล์มีจริงใน dist
2. `{{` — เจอในไฟล์ `.jpg` (binary bytes) ไม่ใช่ใน HTML · ไม่มี template หลุดจริง
3. `/th/vs/${a.orderIdx...}` — เป็น**ซอร์สของ client script** (backtick template ประเมินตอน runtime) ไม่ใช่ href หลุด

## เฟส 3 — Honesty ของระบบที่สร้างเนื้อหาอัตโนมัติ (vs/catalog)

**หน้า vs (`[pair].astro`) — บังคับกฎครบ:**
- เคลม "ดีกว่า" เฉพาะแกนที่ต่าง **≥ 0.3** (บรรทัด 73); ต่างน้อยกว่านั้น → "ไม่มีด้านที่ต่างชัดพอให้เคลม" (บรรทัด 388)
- ไม่ประกาศผู้ชนะเด็ดขาด — ใช้ "นำอยู่ (9.1 ต่อ 8.0) แต่คำตอบขึ้นกับโจทย์" / เคสสูสีใช้ "สูสีกันมาก คำตอบขึ้นกับโจทย์ล้วน ๆ"
- ยอดขายไม่แสดง → "ไม่แสดง" ไม่เดา; ราคาระบุวันเก็บ + "เปลี่ยนได้ทุกวัน"
- ตรวจ runtime: ลิงก์ vs บนหน้าเปรียบเทียบมี guard `Number.isInteger(orderIdx)` — คู่รุ่นคิว (ไม่มี vs page) **ไม่สร้างลิงก์ 404** (ยืนยันแล้ว)

**หน้า catalog (`[slug].astro`) — โปร่งใส:**
- เปิดเผยชัด "อันดับมาจากหน้าค้นหา Shopee เรียงตามยอดขาย ไม่ใช่อันดับที่เราแต่งเอง · ราคาโดยประมาณ" · field ชื่อ `priceApprox`/`soldPerMonth` (ไม่ตีเป็นเลขเป๊ะ)
- บท "อ่านเกมให้:" รายตัว **ดีบั๊กคำเคลมหลอกให้ผู้ซื้อ** เช่น "468,000Pa สูงกว่าเครื่องหมื่นกว่า 20 เท่า เป็นไปได้ยาก อย่าซื้อเพราะตัวเลขนี้" / "กำจัดไร 99.99% เป็นตัวเลขการตลาด" / "ป้ายลดใหญ่ให้เทียบราคาจ่ายจริง ไม่ใช่ราคาก่อนลด"
